TEMPLATE

SITREP VISUAL PROMPT

📋 TEMPLATE: VISUAL SITREP
Buatkan visual SITREP dalam format HTML dengan komponen berikut:

DATA INPUT:
- DTG: [121500Z APR 2026]
- Threat Level: [MEDIUM/HIGH/LOW]
- Period: [last 24 hours]

ENEMY STATUS:
[paste key findings about hostile activity]

FRIENDLY STATUS:
[paste our force readiness/positioning]

KEY EVENTS:
[paste timeline of significant events]

RECOMMENDED ACTIONS:
[paste recommendations]

DESIGN REQUIREMENTS:
1. Header: DTG + Threat Level badge (color-coded)
2. Two-column layout: Enemy (red accent) | Friendly (blue accent)
3. Action items at bottom with green accent
4. Next update time clearly visible
5. Use military color scheme: dark background, tactical colors
6. Total height: fit in single viewport (no scroll)
7. Font: sans-serif, clear hierarchy

OUTPUT: Complete HTML code dengan inline CSS

SITREP LINTAS DOMAIN

4 VARIANT SITREP KOOPSUDNAS

6-element framework (Header/DTG, Level Indicator, Status Card, Friendly Card, Timeline, Actions) tetap universal. Yang berubah: apa yang ditampilkan di tiap card dan stakeholder siapa yang dilayani.

Catatan: Color-coding juga beradaptasi — untuk SAR/Bencana, warna hijau/biru = "progress positif" (sektor aman, shelter tercover), bukan "friendly unit". Pastikan legend eksplisit di tiap variant agar command audience tidak salah baca.
STANDAR

VISUAL QUALITY STANDARDS

✓ DO's

Consistent Color Coding
Red always = threat, Blue always = friendly — no exceptions
Clear Hierarchy
Threat level paling prominent, supporting data lebih kecil
Actionable Output
Setiap visual harus ada "So what?" / recommended action
Single Viewport
Fit dalam satu screen — no scrolling required

✕ DON'Ts

Information Overload
Jangan cram semua data — show summary, detail on request
Decorative Elements
No icons/graphics yang tidak add meaning
Ambiguous Colors
Jangan gunakan warna mirip untuk hal berbeda (red vs orange)
Missing Context
Selalu include DTG, period covered, data source
PRO TIPS

ITERATIVE REFINEMENT

AI jarang menghasilkan visual sempurna di percobaan pertama. Gunakan iterative prompting untuk refine.

Iteration 1: Structure
"Create basic SITREP layout with header, two columns, and action section"
Iteration 2: Visual Polish
"Make the threat level indicator larger and add color-coded badges"
Iteration 3: Content
"Replace placeholder text with actual data: [paste your data]"
Iteration 4: Final Review
"Ensure 3-second rule compliance and add next update timestamp"

TOOL COMPARISON • ALTERNATIVE WORKFLOW

Generate Sitrep Visual dengan Gemini Imagen

Claude tidak bisa generate gambar — Anda hanya bisa pakai Claude untuk describe visual yang diinginkan. Untuk benar-benar menghasilkan sitrep visual, gunakan Gemini Imagen yang built-in di gemini.google.com.

🎨 4-Komponen Prompt Framework

Gunakan struktur yang sama dengan RISEN (Modul 01): disiplin prompt = disiplin output.

SUBJECT: Tactical situational awareness map infographic
STYLE: Modern military briefing, dark navy + amber accent, flat vector
COMPOSITION: Top-down 16:9, labeled zones, legend bottom-right
CONSTRAINT: Generic, no country names, no identifiable aircraft, watermark-ready
✓ Imagen cocok untuk
  • • Sitrep illustrative / generic map view
  • • Infografis statistik threat assessment
  • • Ilustrasi briefing (non-classified)
  • • Icon/symbol set untuk slide
❌ Imagen tidak untuk
  • • Accurate real location map (hallucinate geography)
  • • Identifiable aircraft/equipment spec
  • • Reconstruct real incident (disinformation risk)
  • • Classified facility layout
⚠ OPSEC Gate Wajib

Sebelum generate: jalankan 5 OPSEC checks (Classification, Attribution, Deception, Retention, Watermark). Selalu tambahkan label "AI GENERATED • ILUSTRASI" di output yang keluar organisasi.
