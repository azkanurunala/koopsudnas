IMAGEN 4

STATIC IMAGE GENERATION

Imagen 4 adalah model image generation Google — built-in di gemini.google.com, tidak perlu tool terpisah. Cocok untuk:

✓ Untuk Ops Koopsudnas

  • • Sitrep visual — map-style situational overview
  • • Schematic diagram — force disposition, chain of command
  • • Infografis — statistik threat assessment
  • • Training visuals — ilustrasi untuk briefing
  • • Icon/symbol — unit marker, equipment silhouette

❌ Bukan untuk

  • • Faithful reproduction dari foto nyata (bisa halusinate detail)
  • • Personel identifiable (wajah spesifik)
  • • Real-world deception / disinformation
  • • Sensitive location dengan detail akurat
  • • Classified equipment dengan spec yang leak
⚠ OPSEC Gate sebelum prompt

Setiap request image generation harus lewat pertanyaan: "Jika orang luar melihat output ini, apakah membocorkan informasi operasional?" Kalau YA → jangan generate, atau generalize.

GEMINI CANVAS

INTERACTIVE DASHBOARDS

Canvas adalah fitur live editor di gemini.google.com — Gemini generate HTML/CSS/JS interactive di sebelah chat, Anda bisa iterate tanpa switch tool. Cocok untuk:

📊

Interactive Dashboard

Upload CSV data (threat indicators, sortie counts, equipment status) → "buatkan dashboard dengan chart trend + summary cards"

Output: HTML file self-contained, bisa di-host di local
📝

Live Document

Generate dokumen briefing — formal TNI format, tabel, heading auto. Lebih baik dari chat-to-doc karena live preview.

Output: Google Docs / PDF export langsung
🌐

Web App Prototype

Quick prototype untuk tools internal — threat tracker, intel log viewer, rapid form. No coding skill needed.

Output: functional HTML prototype
🎨

Infographic Builder

Generate infografis dengan SVG — lebih crisp dari Imagen untuk data viz yang butuh teks akurat

Output: SVG export, editable

💡 Canvas vs NotebookLM vs Claude Artifacts

Canvas (Gemini):Live interactive output — user klik, iterate, export. Ideal untuk dashboard & prototype. Built into chat.
NotebookLM:Source-grounded research. Bukan untuk build interactive output — ini tools analisis dokumen.
Claude Artifacts:Serupa Canvas — juga bisa build interactive HTML. Claude Artifacts umumnya lebih kuat di reasoning code-nya, Canvas lebih cepat iterate dengan Gemini ecosystem.
CANVAS TUTORIAL

STEP-BY-STEP WORKFLOW

Dari zero ke interactive intel dashboard dalam 7 langkah

1
Buka Canvas Mode
Di gemini.google.com → klik ikon Canvas (dua kotak) di toolbar sebelah kiri input box. Canvas panel terbuka di kanan layar.
⚠ Pastikan tidak ada data classified di tab browser yang sama
2
Set Context di Chat
Berikan konteks di chat box sebelum minta Canvas generate. Semakin spesifik konteks → semakin relevan output.
"Saya perlu dashboard intel untuk monitoring sortie count harian. Data: [paste data]. Buat interactive HTML."
3
Generate Pertama
Gemini generate HTML/CSS/JS dan render langsung di Canvas panel kanan. Anda bisa lihat hasilnya real-time sambil kode ditulis.
4
Inspect & Interact
Klik elemen di Canvas preview — filter, sorting, hover tooltip. Kalau ada yang tidak benar atau kurang, catat spesifik apa yang mau diubah.
5
Iterate via Chat
Gunakan chat untuk refine — tidak perlu re-prompt dari awal. Canvas ingat context dari iterasi sebelumnya.
"Ubah warna header jadi navy dark, tambahkan filter dropdown per tanggal, pindahkan summary cards ke atas chart"
6
Preview Full Screen
Klik ikon expand di Canvas panel untuk full-screen preview. Test semua interaksi: click, hover, filter, sort sebelum export.
7
Export & Deploy
Pilih export format sesuai kebutuhan:
• Download HTML → standalone file, bisa buka tanpa internet
• Export to Google Docs → kolaborasi + komentar
• Share link → view-only, cocok untuk distribusi ke komandan
PRO TIP: Prompt Template Canvas Dashboard
Buat interactive HTML dashboard untuk [TOPIK]. Data: [paste atau deskripsi data]. Tampilkan: [list komponen — chart, table, summary cards]. Warna scheme: dark navy, accent [warna]. Bahasa Indonesia. Self-contained single HTML file, tidak perlu CDN.
VEO

VIDEO GENERATION

🎬

Veo 3 — Google's Video Model

Generate video 8-60 detik dari text prompt atau image-to-video. Audio integration (dialogue, ambient) built-in sejak Veo 3.

⚠ Tersedia di Gemini Advanced (paid tier) dan AI Studio dengan quota terbatas.

✓ Legitimate Ops Use

  • • Training video — generic battlefield scenario untuk doctrine class
  • • Public affairs clip — hari TNI AU, ceremonial b-roll
  • • Concept visualization — show komandan "kira-kira seperti ini" untuk konsep operasi baru
  • • Recruitment material — visual appeal untuk generasi muda

❌ Red Flags

  • • Deepfake tokoh politik/militer — dilarang by policy
  • • Reconstruct insiden nyata — bisa dianggap disinformation
  • • Fake news narrative video — etis dan legal problem
  • • Classified visualization — equipment/facility yang bocor

🎯 Contoh Prompt Veo (acceptable)

SUBJECT: Generic military aircraft silhouette at sunset, training sortie returning to base
CAMERA: Slow aerial tracking shot, cinematic, 24fps, golden hour lighting
MOOD: Professional, inspiring, disciplined — not action-movie sensationalized
DURATION: 8 seconds, loop-ready composition
CONSTRAINT: No specific country markings, no identifiable squadron insignia, no real-world locations recognizable, no weapons shown. Training/recruitment material only.
TEKNIK LANJUTAN

ITERATIVE PROMPTING

First output adalah starting point, bukan finish line — ini cara iterate menuju hasil professional

Pattern: Generate → Critique → Refine

G
Generate (Round 1)
Pakai prompt 4-komponen. Generate 4 variant. Pilih base terbaik — jangan terlalu lama memilih, ini hanya starting point.
C
Critique Spesifik
Identifikasi 2-3 masalah konkret. Hindari feedback vague seperti "kurang bagus" — tulis apa tepatnya yang salah.
❌ "Kurang profesional"
✓ "Warna background terlalu terang, teks legend terlalu kecil, tidak ada north indicator"
R
Refine Prompt
Tambahkan correction ke prompt. Tambahkan ke bagian CONSTRAINT atau STYLE sesuai masalah. Generate ulang hanya elemen yang bermasalah jika bisa.

Vocabulary Refinement

Kata-kata ini di prompt image menghasilkan perubahan yang predictable:

UNTUK GAYA / ESTETIKA
editorial quality highly detailed minimalist clean flat vector art isometric view
UNTUK SKEMA WARNA
monochromatic dark mode palette amber accent on navy desaturated muted
UNTUK KOMPOSISI
rule of thirds centered symmetrical full bleed background negative space heavy
UNTUK TEKS DI GAMBAR
readable labels callout lines with text monospace font labels no text in image

Contoh Iterasi 3-Ronde: Sitrep Infografis

ROUND 1 — BASELINE
Prompt minimal: "tactical map infographic, dark theme, military style"
Output: generic, warna terlalu cerah, tidak ada detail militer, teks tidak readable
ROUND 2 — REFINED
Tambah: "flat vector, navy #050810 background, amber accent, labeled sectors A-D, north indicator, monospace labels, no photorealism"
Output: jauh lebih baik, tapi legend masih kurang jelas dan tidak ada scale indicator
ROUND 3 — FINAL
Tambah: "bottom-right legend panel with 4 icon types, scale bar bottom-left in km, editorial quality, print-ready resolution"
Output: briefing-ready, semua elemen ada, konsisten dengan standar visual militer
OPSEC CRITICAL

VISUAL GENERATION RISKS

Visual generation tools lebih berbahaya dari text generation untuk OPSEC. Output visual bisa viral, di-screenshot, dan sulit di-retract. Sebelum generate apapun:

1
Classification Check
Apakah ada detail dalam visual yang jika dilihat publik akan membocorkan klasifikasi operasional? Contoh: aircraft spec, sensor package, facility layout.
2
Attribution Check
Jika visual keluar dari organisasi, apakah bisa diattribute ke Koopsudnas/TNI AU? Jika ya, pastikan ada watermark/disclaimer "ILUSTRASI — AI GENERATED".
3
Deception Check
Apakah visual bisa digunakan orang jahat untuk deceive atau spread disinformation? Terutama video Veo — generic sunset aircraft OK, tapi "reenactment" insiden spesifik berbahaya.
4
Retention Check
Output visual akan di-store di Google cloud. Cek kebijakan retention + data handling akun Anda. Data sensitif → jangan generate di akun personal.
5
Watermark Mandatory
Gemini menyisipkan SynthID watermark otomatis di semua output Imagen/Veo. Jangan crop/edit untuk hilangkan. Sebagai operator, tambahkan label "AI GENERATED • ILUSTRASI • [TANGGAL]" di setiap visual yang keluar organisasi.
DECISION TREE: BOLEH GENERATE?
✓ Generic, tidak ada country/unit/spec spesifik → BOLEH
? Ada konteks operasional tapi masih bisa digeneralize → GENERALIZE DULU, lalu generate
✗ Classified detail tidak bisa dihapus tanpa kehilangan fungsi → JANGAN GENERATE
HANDS-ON EXERCISE

VISUAL INTEL PACKAGE

20 Menit • Produce 2 visual product dari skenario yang sama

📋 Skenario Tasking

Anda diminta menyiapkan visual package untuk briefing "Situasi Udara Indo-Pasifik Q1 2026" kepada Komandan. Package harus terdiri dari: (1) Infografis statis sitrep menggunakan Imagen, dan (2) Interactive summary dashboard menggunakan Canvas. Semua konten harus lolos OPSEC 5-checks.

1
OPSEC Pre-Check (2 min)

Sebelum mulai, tentukan: apa saja yang tidak boleh ada di visual ini? Tulis 3-5 constraint OPSEC spesifik untuk skenario briefing ini. Ini akan masuk ke CONSTRAINT block semua prompt Anda.

2
Imagen — Sitrep Infografis (6 min)

Tulis prompt 4-komponen (Subject + Style + Composition + Constraint). Generate 4 variant. Pilih 1 baseline. Critique 2 masalah spesifik. Lakukan 1 iterasi refinement. Target: visual yang siap masuk briefing deck.

Hint: Template I-01 adalah starting point yang baik — adapt dan personalisasi
3
Canvas — Interactive Dashboard (8 min)

Buka Canvas mode. Tulis prompt spesifik: dashboard dengan minimal 3 elemen (summary cards, chart, dan tabel). Gunakan data placeholder (angka generic). Iterate 1 kali untuk improve satu elemen yang kurang. Download sebagai HTML file.

Hint: Template Canvas Dashboard di slide sebelumnya — copy dan adapt
4
Final OPSEC Audit (2 min)

Jalankan 5 OPSEC checks untuk kedua output. Apakah lolos semua? Jika ada yang gagal, iterate prompt sekali lagi untuk generalize.

5
Reflection (2 min)

Untuk skenario briefing komandan ini: kapan Imagen lebih tepat vs Canvas? Mana yang lebih efisien untuk produksi regular di Koopsudnas? Ada gap capability yang masih butuh tools lain?

TARGET OUTPUT
1 infografis Imagen (sitrep) + 1 HTML dashboard Canvas
SUCCESS CRITERIA
Kedua output lolos 5 OPSEC checks dan siap digunakan sebagai draft briefing material

WRAP UP

KEY TAKEAWAYS

1
3 Tools, 3 Purposes
Imagen = ilustrasi / infografis statis • Canvas = interactive HTML / SVG • Veo = video
2
4-Part Prompt Mandatory
Subject + Style + Composition + Constraint — tanpa 4 komponen ini output tidak reproducible
3
OPSEC 5-Checks Wajib
Classification, Attribution, Deception, Retention, Watermark — semua visual keluar organisasi wajib lolos 5 checks ini
4
Iterative, Bukan Sekali
First output adalah starting point. Critique spesifik → refine prompt → iterate. 2-3 ronde untuk hasil briefing-ready.
5
Canvas = Interactive Intelligence
Untuk dashboard yang bisa di-filter komandan real-time, Canvas adalah tool terbaik. Download sebagai standalone HTML — tidak perlu internet untuk present di briefing room.
