KONSEP DASAR

APA ITU CANVAS?

💬

Mode Chat (biasa)

  • ✗ Respons AI terpisah dari dokumen kerja Anda
  • ✗ Setiap revisi = prompt baru + copy-paste manual
  • ✗ Tidak bisa edit bagian tertentu secara terisolasi
  • ✗ Tidak ada export langsung
📋

Mode Canvas

  • ✓ Dokumen hidup di panel kanan, chat di kiri
  • ✓ Highlight teks → instruksi AI → update inline
  • ✓ Revisi satu section tanpa ganggu bagian lain
  • ✓ Export ke Google Docs / copy / print PDF
🎯
Analogi sederhana
Canvas = Google Docs dengan AI embedded. Anda tidak perlu bolak-balik copy-paste antara ChatGPT/Gemini dan Word. Dokumen dan AI ada dalam satu layar, berkolaborasi secara langsung.
HOW TO START

CARA BUKA CANVAS

1
Buka gemini.google.com
Login dengan akun Google. Pastikan menggunakan Gemini 2.5 Flash atau Pro (bukan versi lama).
2
Klik "Canvas" di sidebar kiri atau Tools
Ikon berbentuk dokumen/papan. Atau ketik langsung di chat: "Buat Canvas untuk SITREP Hanud"
3
Layar split: Chat (kiri) + Dokumen (kanan)
Panel kanan menampilkan dokumen aktif. Panel kiri tetap sebagai chat untuk instruksi. Anda bisa scroll panel kanan secara independen.
4
Berikan instruksi pertama
Ketik di chat: template yang diinginkan, konteks, format. AI akan populate dokumen di panel kanan secara langsung.
5
Highlight → Edit Section Tertentu
Di panel kanan, select bagian teks → klik ikon AI yang muncul → berikan instruksi spesifik untuk bagian itu. Bagian lain tidak tersentuh.

TEKNIK LANJUTAN

SECTION-LEVEL EDITING

Keunggulan Canvas yang paling sering dipakai Pamen: revisi satu bagian dokumen tanpa rewrite keseluruhan. Berikut teknik lengkapnya:

Teknik 1 — Highlight + Inline Prompt
Select teks di panel kanan → ikon pensil AI muncul → ketik instruksi → AI update hanya bagian itu.
[Highlight: paragraph "Rekomendasi Tindakan"]

Perkuat rekomendasi ini dengan 2 opsi COA konkret:
COA 1: tindakan preventif (response time 15 menit)
COA 2: tindakan agresif (scramble immediate)
Tambahkan risk/benefit masing-masing COA dalam 1 kalimat.
Teknik 2 — Chat Instruction ke Bagian Bernama
Dari panel chat, instruksikan secara eksplisit bagian mana yang direvisi tanpa highlight.
Perbarui HANYA section "Intelligence Summary" dengan tambahan:
- Confidence level untuk setiap sumber: HIGH/MEDIUM/LOW
- Flag satu data gap yang paling kritikal
Jangan ubah section lain di dokumen ini.
Teknik 3 — Minta AI Tambahkan Tabel
AI bisa insert tabel langsung di Canvas — sangat berguna untuk readiness matrix atau resource allocation SAR.
Setelah section "Kekuatan Udara", tambahkan tabel readiness dengan kolom:
| Satuan | Aset | Serviceable | FMC% | Crew Ready | Status |
Isi dengan data placeholder yang realistis untuk 3 skadron.
Format tabel: Markdown table agar bisa di-export ke Docs.
PROMPT LIBRARY

TEMPLATE CANVAS

Buat Canvas untuk SITREP Hanud periode 24 jam terakhir.

Struktur dokumen:
1. EXECUTIVE SUMMARY (3-4 kalimat, tone komandan)
2. SITUASI UDARA — kontak aktif, status IFF, anomali
3. POSTUR HANUD — aset siap, radar coverage, satuan on-alert
4. THREAT ASSESSMENT — level ancaman + confidence (HIGH/MED/LOW)
5. INTELLIGENCE GAPS — apa yang belum diketahui
6. REKOMENDASI — 3 tindakan prioritas

Skenario: Satu kontak unidentified terpantau masuk FIR Ujung Pandang dari arah NE, kecepatan 480 knot, tidak merespons IFF. Kontak hilang setelah 12 menit.

Gunakan placeholder [WAKTU], [KOORDINAT], [CALLSIGN] untuk data yang akan diisi manual. Format: siap cetak untuk briefing komandan.
Buat Canvas untuk SAR Action Plan operasi CENDRAWASIH RESCUE.

Skenario: Pesawat sipil (ATR-72) hilang kontak di Pegunungan Bintang, Papua. Last contact H-3 jam. Cuaca buruk (ceiling 500ft, vis 1km). Koordinasi multi-agency: TNI AU (helikopter SAR), Basarnas (ground team), Kemenhub (ATC data), Pemda (logistik).

Struktur dokumen:
1. SITUATION OVERVIEW — kronologi singkat
2. SEARCH AREA PRIORITY — 3 area berdasarkan last known position + ETOPS deviation
3. ASSET ASSIGNMENT — tabel: Aset | Instansi | Wilayah | Jam Berangkat | Status
4. COMMAND & COORDINATION — siapa mengomando apa, jalur komunikasi
5. WEATHER CONSTRAINTS — go/no-go criteria per area
6. TIMELINE H+0 s/d H+72
7. CONTINGENCY — jika cuaca tidak membaik dalam 24 jam

Bahasa: formal militer Indonesia. Siap untuk briefing Pangkoopsud.
Buat Canvas untuk Doctrine Note: "Update Postur IADS Koopsudnas menghadapi Threat Drone Swarm".

Struktur:
1. PENDAHULUAN — latar belakang, mengapa doktrin perlu diupdate
2. TINJAUAN ANCAMAN — karakteristik drone swarm modern, gap deteksi radar konvensional
3. REVIEW DOKTRIN EXISTING — apa yang masih relevan, apa yang obsolete
4. LESSONS LEARNED — dari exercise ADEX terbaru + studi kasus regional (generic)
5. REKOMENDASI PERUBAHAN DOKTRIN — 5 perubahan prioritas
6. IMPLIKASI ALUTSISTA — sistem C-UAS apa yang perlu diakuisisi/diupgrade
7. KESIMPULAN

Tone: akademik-militer. Panjang: ±4 halaman A4. Catatan: ini untuk internal working draft, bukan publikasi final.
BEST PRACTICE

TIPS & PITFALLS

✓ TIPS YANG BEKERJA
  • ✓
    Mulai dengan struktur dulu — minta AI buat kerangka kosong, baru isi section per section
  • ✓
    Satu instruksi, satu perubahan — jangan campur beberapa instruksi dalam satu prompt Canvas
  • ✓
    Sebutkan nama section — "update HANYA bagian Executive Summary" lebih aman dari "perbarui bagian atas"
  • ✓
    Export sering — simpan versi ke Google Docs setiap milestone penting untuk audit trail
✗ PITFALLS YANG SERING TERJADI
  • ✗
    Instruksi ambigu — "perbaiki dokumennya" tanpa spesifikasi apa yang diperbaiki → AI mungkin rewrite seluruh dokumen
  • ✗
    Tidak verifikasi output — AI bisa "mengarang" data placeholder yang terlihat nyata. Selalu ganti dengan data aktual sebelum distribusi.
  • ✗
    Dokumen terlalu panjang — Canvas paling efektif untuk dokumen 1-5 halaman. Dokumen 20+ halaman lebih cocok pakai NotebookLM.
  • ✗
    Lupa OPSEC — jangan masukkan data classified, callsign nyata, atau koordinat aktual ke Canvas (cloud-based tool).
DECISION MATRIX

CANVAS vs CHAT vs NOTEBOOKLM

KRITERIA Chat Gemini Canvas NotebookLM
TASK TYPE Q&A, quick analysis Draft & iterate dokumen Synthesis dari banyak dokumen
INPUT Teks, gambar, audio Instruksi + template Upload PDF/dokumen
ITERASI Copy-paste manual Inline, section-level Tanya-jawab berbasis sumber
EXPORT Copy text only Google Docs, PDF Audio overview, cite snippet
BEST FOR Tanya cepat, research SITREP, briefing, SOP Intel synthesis, doktrin review
Rule of thumb: Kalau tujuan akhirnya adalah dokumen yang bisa di-print dan ditandatangani → pakai Canvas. Kalau tujuannya adalah menjawab pertanyaan dari banyak dokumen yang sudah ada → pakai NotebookLM.
HANDS-ON

LATIHAN 20 MENIT

🎯 Setup (2 menit)

Buka gemini.google.com → aktifkan Canvas (klik "Canvas" di sidebar atau Tools). Anda akan membuat SITREP multi-domain lengkap dalam satu dokumen Canvas, iterasi section per section.

1
Buat SITREP Hanud dasar (6 menit)
DOMAIN A

Gunakan prompt C-01 dari Prompt Library (slide sebelumnya). Biarkan AI generate draf pertama. Baca hasilnya — jangan edit dulu.

2
Tambah section SAR (6 menit)
DOMAIN B

Dari panel chat, instruksikan: "Tambahkan section baru setelah Threat Assessment: 'OMSP — SAR Status' untuk laporan satu insiden aircraft emergency di Papua. Jangan ubah section lain." Amati bagaimana Canvas hanya menambahkan section baru.

3
Revisi Executive Summary (3 menit)
ITERASI

Highlight section Executive Summary → instruksi: "Persingkat menjadi 2 kalimat, tone harus lebih assertive untuk level Pangkoopsud." Bandingkan sebelum dan sesudah.

4
Export ke Google Docs (3 menit)
EXPORT

Klik tombol "Export" di Canvas → "Open in Google Docs". Lihat format, rapikan judul, tambahkan header instansi. Ini adalah workflow akhir sebelum distribusi ke satuan.
