WHY MULTI-TOOL

NO SINGLE TOOL WINS

Operator yang matang tidak loyal ke 1 tool. Mereka routing tiap task ke tool yang paling cocok, handoff context dengan disiplin, dan tahu kapan tidak pakai AI sama sekali. Modul 06 adalah tentang membangun tool-switching discipline.

📚 NotebookLM
Sourced synthesis, zero hallucination, audio brief
Best for: ingest 20+ docs, citation wajib
✓ Source-grounded, citable
✓ Audio overview
✗ No live search
✗ No image generation
🧠 Claude
Deep reasoning, ethical nuance, narrative quality
Best for: COA analysis, commander brief draft
✓ Best strategic reasoning
✓ Nuanced judgment
✗ No native image gen
✗ No real-time search (base)
🔍 Gemini
Search grounding, multimodal, Imagen/Canvas/Veo
Best for: real-time OSINT, visual generation
✓ Live search grounding
✓ Imagen/Canvas/Veo
✗ Less nuanced reasoning
✗ Source attribution weaker
🎯 Prinsip Dasar Workflow
  • 1. Ground first. NotebookLM atau Gemini search → kumpulkan fakta dengan citation sebelum reasoning.
  • 2. Reason second. Claude untuk analisa strategis & nuanced judgment.
  • 3. Visualize last. Gemini Imagen/Canvas untuk output visual.
  • 4. Human gate always. Komandan = final approver, bukan AI.
CAPSTONE SCENARIO

SKENARIO OPERASIONAL

🎯

Tasking

Anda adalah Pamen di Pusintelsudnas. Komandan Koopsudnas meminta "briefing package lengkap 30 menit" tentang perkembangan kapabilitas UAV regional dalam 6 bulan terakhir — mencakup threat indicators, implications untuk postur pertahanan udara, dan rekomendasi COA.

Waktu persiapan: 90 menit. Output harus terdiri dari: (a) executive summary tertulis, (b) 1-2 visual slide, (c) audio brief 10 menit untuk didengar komandan di kendaraan, (d) appendix dengan sumber.

⚠ Constraint
  • • Hanya gunakan sumber OSINT publicly available (no classified)
  • • Semua klaim harus traceable ke sumber
  • • Visual harus generic, lulus 5 OPSEC checks (Sesi 6.3)
  • • Audio brief tone profesional militer, bukan enthusiast
📦 Deliverable
  • • Executive summary — 2 halaman, formal TNI
  • • 1-2 visual — Imagen atau Canvas
  • • Audio brief 10 menit — NotebookLM
  • • Appendix citations — semua sumber
TIMELINE ALLOCATION
Phase 1 NotebookLM: 30 min
Phase 2 Claude: 25 min
Phase 3 Gemini: 20 min
Phase 4 Package: 15 min
4-PHASE PIPELINE

WORKFLOW END-TO-END

P1
PHASE 1 • 30 MENIT • NOTEBOOKLM

Ingest & Ground

INTEL FUSION
1.
Create notebook "UAV_REGIONAL_Q4_2025-Q1_2026". Upload 15-25 sources: Jane's, IISS, Defense News, CSIS reports, regional military analysis blogs.
2.
RISEN query terstruktur: "Extract UAV capability developments by country, categorized by: acquisition, doctrinal shift, operational deployment, incident/near-incident."
3.
Generate briefing doc di Studio panel. Review, spot-check 3-5 citations.
4.
Export output — copy briefing doc ke clipboard / markdown file untuk hand-off ke Phase 2 menggunakan Template H-01.
🎯 Deliverable Phase 1: Markdown briefing doc dengan fact claims + inline citations. Ini adalah raw intel baseline, belum ada analisis strategis.
P2
PHASE 2 • 25 MENIT • CLAUDE

Strategic Reasoning & COA

ANALYSIS
1.
Paste H-01 handoff (Template dari slide berikutnya) ke Claude. Set role: "Senior intelligence analyst Koopsudnas".
2.
Deep thinking prompt: "Think step-by-step: what are the 3 most significant threat indicators? What baseline deviations matter? What are the implications for Indonesian air defense posture?"
3.
COA analysis: "Generate 3 COAs untuk respond ke threat: conservative, balanced, aggressive. Include pros/cons/resource requirements untuk tiap COA."
4.
Draft executive summary — Claude tulis versi awal commander's brief dalam bahasa formal TNI, max 2 halaman. Export menggunakan Template H-02.
🎯 Deliverable Phase 2: Executive summary draft + 3 COA analysis. Reasoning sudah ada, tinggal validasi dan enhance.
P3
PHASE 3 • 20 MENIT • GEMINI

Search Grounding & Visual

GROUNDING + VISUAL
1.
Fact check via search grounding. Buka Gemini, enable "Grounding with Google Search". Paste 3-5 claim paling kritikal dari Phase 2, minta: "verify this claim against latest public reporting, cite sources".
2.
Fill last-mile gaps — kalau ada update 48 jam terakhir yang NotebookLM missed (karena source upload), Gemini akan catch via live search.
3.
Generate sitrep visual via Imagen. Prompt 4-komponen (Sesi 6.3). OPSEC: generic, no country names. Sebagai ilustrasi slide briefing.
4.
Build dashboard via Canvas (optional). Kalau punya CSV data indicators, minta Gemini Canvas build interactive viz untuk komandan klik-klik. Export menggunakan Template H-03.
🎯 Deliverable Phase 3: Verified claims + 1-2 visual slide (Imagen) + dashboard opsional (Canvas). Ini adalah presentation assets.
P4
PHASE 4 • 15 MENIT • NOTEBOOKLM

Package & Audio Brief

FINAL PACKAGE
1.
Kembali ke NotebookLM. Add executive summary (Phase 2) dan verified claims (Phase 3) sebagai sources baru di notebook.
2.
Generate Audio Overview dengan custom instruction profesional: "Professional military tone, Indonesian language, focus on operational implications, 10 minutes."
3.
QA sebelum share — dengar 2 menit pertama, pastikan tone tepat, tidak sensasional, akurat. Jika tidak oke, regenerate dengan instruction yang lebih spesifik.
4.
Assemble package: (a) PDF executive summary, (b) slide visual, (c) MP3 audio brief, (d) appendix citations. Tag file: "UAV_REGIONAL_BRIEF_2026-04-13_v1".
🎯 Deliverable Phase 4: Complete intel package — executive summary + visual + audio + appendix. Ready untuk komandan.

# VERIFICATION REQUEST — Phase 2 → Phase 3 ## Context This is an intelligence analysis verification request. FILL: pilih 3-5 klaim yang paling kritikal dari analisis Claude — prioritaskan klaim yang akan jadi basis rekomendasi COA
Generate an audio overview briefing with the following specific requirements: AUDIENCE: Komandan Koopsudnas (senior military officer, familiar with doctrine, time-constrained) TONE: Professional Indonesian military language (Bahasa Indonesia formal). Not enthusiast, not academic — operational and direct. Equivalent to how a trusted staff officer would brief a commander. STRUCTURE (follow exactly): 1. Situasi Umum (2 menit) — what is happening, factual, no editorializing 2. Perkembangan Kritis (3 menit) — 3 most significant changes since last brief 3. Implikasi Operasional (3 menit) — what this means for our posture 4. Opsi Tindak Lanjut (2 menit) — COA options, not recommendations LENGTH: Exactly 10 minutes (approximately 1,400-1,600 Indonesian words spoken at briefing pace) AVOID: - Speculative language ("might", "could", "perhaps") - Sensationalist framing - Overly technical jargon without explanation - Opinions not grounded in the source documents

DEEP DIVE: GEMINI PHASE 3

DEEP RESEARCH WALKTHROUGH

Satu-satunya tool yang bisa do live multi-source research dalam 10 menit — tapi ada cara benar dan salah menggunakannya

Kapan Pakai Gemini Deep Research

✓
Gap-filling setelah NotebookLM
NotebookLM hanya punya data sampai kapan sumber diupload. Deep Research tangkap 24-48 jam terakhir.
✓
Cross-verify klaim kritis
Sebelum COA masuk ke brief komandan, pastikan factual foundation kuat via live search.
✓
Broad landscape scan awal
Kalau tidak punya 15-25 sumber untuk NotebookLM, Deep Research bisa jadi Phase 1 alternatif — tapi output tidak ada source guarantee.
✗
Jangan sebagai pengganti NotebookLM
Deep Research tidak bisa ingest dokumen internal. Untuk source-grounded citation dari dokumen spesifik → tetap NotebookLM.

Step-by-Step: Jalankan Deep Research

1
Aktifkan Deep Research Mode
Di gemini.google.com → klik ikon search/earth di toolbar bawah → pilih "Deep Research". Hanya tersedia di Gemini Advanced.
2
Tulis Research Query
Query harus spesifik dengan timeframe dan scope. Lihat template di bawah. Gemini akan tanya konfirmasi research plan sebelum mulai.
3
Review Research Plan
Gemini presentasi rencana penelitian (berapa sources, angle apa). Adjust scope kalau terlalu luas atau terlalu sempit sebelum approve.
4
Wait & Monitor
Deep Research mengambil 3-10 menit. Jangan interrupt. Hasil hadir sebagai structured report dengan source links.
5
Verify Top Sources
Click 2-3 source links di report — pastikan bukan agregasi blog yang unreliable. Prioritaskan Jane's, IISS, reputable defense media.

Template Query Deep Research (untuk Phase 3)

Conduct deep research on: [TOPIK SPESIFIK — contoh: "UAV acquisition and operational deployment in Southeast Asia, October 2025 - March 2026"] Focus areas: 1. Confirmed acquisitions, deliveries, or procurement contracts (with dates and source citations) 2. Doctrinal or deployment changes (new units, new roles, new operational areas) 3. Incidents or near-incidents involving UAV assets in the region 4. Statements by defense officials regarding UAV capability development Sources to prioritize: Jane's Defence, IISS Military Balance, Defense News, Reuters Defense, AP, official government defense ministry statements Exclude: blogs, aggregators, social media unless corroborated by primary source Output format: structured report with inline citations, organized by country/actor, with a "Key Developments Summary" at the top
SCOPE TIP
Lebih spesifik = hasil lebih actionable. "UAV Southeast Asia 2026" terlalu luas — add: country subset, capability type, timeframe.
QUALITY CHECK
Setelah hasil keluar, cek: apakah sources yang di-cite bisa di-verify? Klik 2-3 link random untuk konfirmasi bukan AI-generated content.
FALLBACK
Kalau Deep Research tidak tersedia (kuota habis) → gunakan Gemini standard dengan Search Grounding enable + multi-turn manual search.
HANDOFF DISCIPLINE

AVOID CONTEXT LOSS

Bahaya utama multi-tool: context bocor, fact drift, dan fatigue. Mitigate dengan disiplin handoff.

1️⃣
Always use structured export
Handoff antar tool harus dalam format terstruktur (Markdown, JSON) — bukan copy-paste percakapan. Markdown dengan heading jelas = konteks transfer clean. Gunakan Templates H-01, H-02, H-03.
2️⃣
Preserve citation chain
Setiap klaim yang di-forward dari Phase 1 → Phase 2 → Phase 4 harus tetap punya citation. Kalau hilang di tengah jalan = claim tidak bisa defend di hadapan komandan.
3️⃣
Version everything
v1 draft NotebookLM, v2 Claude analysis, v3 Gemini enhanced, v4 final. Jangan overwrite — kalau ada bug di v3, masih bisa rollback ke v2.
4️⃣
Human gate at every phase boundary
Jangan rantai AI ke AI tanpa human review. Setelah tiap Phase, operator harus spot-check output sebelum masuk ke tool berikutnya. Otherwise, errors compound.
5️⃣
Track time-to-retreat
Kalau total workflow sudah lewat 2x estimated time dan belum selesai — itu red flag. Mungkin scope terlalu besar atau tool salah routing. Retreat, re-plan, bukan push through.
ANTI-PATTERNS

COMMON FAILURE MODES

🔥 Tool overload

Pamen pakai 3 tools untuk task yang sebenarnya cukup 1 tool. Hasil: lelah, tidak konsisten, tidak lebih baik.

Fix: mulai dari 1 tool, tambahin hanya kalau gap jelas.

🔥 Citation amnesia

Klaim di Phase 1 punya citation, tapi pas ter-copy ke Claude di Phase 2, citation hilang. Di Phase 4, tidak ada source-nya lagi.

Fix: always handoff in markdown dengan footnote preserved. Gunakan Template H-01.

🔥 Fact drift

Setiap tool "slightly improve" claim — sampai di Phase 4, claim sudah berbeda dari fact aslinya.

Fix: validate di Phase 3 dengan Gemini search grounding; lock facts di Phase 2 onwards.

🔥 Tool religion

"Saya cuma percaya Claude" atau "NotebookLM tidak bisa dipercaya" — bias pribadi bocor ke routing.

Fix: routing berdasarkan task properties, bukan favoritism.

🔥 OPSEC erosion

Data sensitif ter-upload ke akun personal "karena buru-buru". Multi-tool = multi surface area, risiko leak bertambah.

Fix: gunakan akun institusional di semua tools, audit retention policy.

🔥 Human gate skip

AI ke AI langsung, operator jadi "AI orchestrator" yang tidak baca output. Kalau salah, tidak ada yang catch.

Fix: minimal 30 detik spot-check tiap phase boundary.

b. Minta: top 3 threat indicators + COA analysis (conservative/balanced/aggressive) + executive summary.
c. Review output: apakah citations dari Phase 1 masih terjaga? Apakah ada klaim baru yang tidak ada sumbernya (flag dengan [VERIFY])?

Gemini — Verify & Visualize (20 min)
55:00 – 75:00

NotebookLM — Package & Audio (15 min)
75:00 – 90:00
a. Kembali ke notebook NotebookLM. Add "v3_verified.md" sebagai source baru.
b. Generate Audio Overview menggunakan Template H-03 di Custom Instructions. Generate + dengar 2 menit pertama untuk QA tone.
c. Assemble package akhir: executive summary PDF + visual Imagen + audio MP3 + appendix citations.
d. Reflection: Apa yang berjalan lancar? Di mana phase paling time-consuming? Tool mana yang memberikan leverage terbesar untuk kasus ini?

KNOWLEDGE CHECK

QUIZ SESI 6.9

4 pertanyaan • Pilih jawaban terbaik

SOAL 1 / 4

Dalam 4-phase multi-tool pipeline, mengapa NotebookLM digunakan di Phase 1 sebelum Claude?

✓ Ground first = kumpulkan fakta dengan citation. Claude reasoning di Phase 2 sebaiknya berbasis fact, bukan generate dari knowledge internal-nya saja.

SOAL 2 / 4

"Citation amnesia" dalam multi-tool workflow adalah ketika...

✓ Citation amnesia = citations hilang di tengah pipeline. Mitigasi: selalu handoff dalam markdown dengan citations dipreserve, gunakan Template H-01.

SOAL 3 / 4

Apa keunggulan utama Gemini (dengan Search Grounding) dibanding NotebookLM untuk Phase 3 verification?

✓ NotebookLM hanya bisa access dokumen yang diupload. Gemini + Search Grounding bisa fill gap dengan live web search — critical untuk intel currency.

SOAL 4 / 4

Apa format terbaik untuk context handoff antar tool dalam multi-tool pipeline?

✓ Structured markdown = konteks transfer clean. Screenshot tidak bisa di-parse AI. Chat history penuh = noise. Ringkasan lisan = menghilangkan nuance dan citations.
