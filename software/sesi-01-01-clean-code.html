KONSEP 1/3

NAMING ADALAH
DOKUMENTASI

Nama yang baik menghilangkan kebutuhan komentar. Aturan sederhana: kalau butuh komentar untuk menjelaskan variabel/function, rename saja.

❌ BURUK
// list of active users from last 30 days
const d = getUsers()
  .filter(u => u.a && daysSince(u.lt) < 30);

Nama d, u, a, lt — butuh komentar biar paham.

✅ BAIK
const activeUsers = getUsers()
  .filter(user =>
    user.isActive &&
    daysSince(user.lastLoginAt) < 30
  );

Tidak butuh komentar. Intent terbaca langsung dari kode.

📝 NAMING RULES
  • • Boolean: pakai is, has, should → isActive, hasPermission
  • • Function: verb phrase → fetchUser(), calculateTotal()
  • • Collection: plural noun → users, orderItems
  • • Constant: SCREAMING_SNAKE_CASE → MAX_RETRY_COUNT
KONSEP 2/3

FUNCTION:
FOKUS KE SATU HAL

Satu function — satu level abstraksi, satu alasan untuk berubah. Kalau kamu bisa extract sub-function dengan nama yang bermakna, lakukan.

❌ DO-EVERYTHING FUNCTION
function processOrder(order) {
  // validate
  if (!order.items || order.items.length === 0) throw new Error('empty');
  let total = 0;
  for (const item of order.items) total += item.price * item.qty;
  // apply discount
  if (order.couponCode) { /* ... 20 lines ... */ }
  // save
  db.orders.insert({ ...order, total });
  // send email
  emailService.send(order.email, 'Order confirmed', `Total: ${total}`);
  return total;
}
✅ DECOMPOSED
function processOrder(order) {
  validateOrder(order);
  const subtotal = calculateSubtotal(order.items);
  const total = applyDiscount(subtotal, order.couponCode);
  const saved = saveOrder({ ...order, total });
  sendConfirmationEmail(saved);
  return total;
}

Setiap baris punya satu level abstraksi. Testable, readable, replaceable.

KONSEP 3/3

ABSTRACTION:
WAIT UNTIL RULE OF 3

DRY (Don't Repeat Yourself) bukan larangan untuk menulis kode serupa dua kali. Premature abstraction lebih mahal daripada duplication. Aturan praktis: abstract setelah duplikasi ketiga, dan hanya jika ketiga use-case benar-benar punya reason-to-change yang sama.

1️⃣

First occurrence

Write the code. Don't anticipate.

2️⃣

Second occurrence

Copy-paste ok. Note the similarity mentally.

3️⃣

Third occurrence → extract

Sekarang kamu punya 3 data points tentang apa yang benar-benar invariant vs. what varies. Abstraction yang kamu bikin akan lebih akurat daripada kalau extract di occurrence pertama.

SIDE-BY-SIDE

BAD CODE VS
CLEAN CODE

❌ BAD
function calc(a, b, t) {
  if (t === 1) return a + b;
  if (t === 2) return a - b;
  if (t === 3) return a * b;
  if (t === 4) return a / b;
}
calc(10, 5, 3); // 50? WTF is 3?
  • • Nama tidak bermakna
  • • Magic numbers
  • • Caller tidak tahu intent
  • • Tanpa error handling
✅ CLEAN
const Operation = {
  ADD: 'add', SUBTRACT: 'subtract',
  MULTIPLY: 'multiply', DIVIDE: 'divide'
};

function calculate(left, right, op) {
  switch (op) {
    case Operation.ADD: return left + right;
    case Operation.MULTIPLY: return left * right;
    // ...
  }
}
calculate(10, 5, Operation.MULTIPLY);
  • • Self-documenting names
  • • No magic numbers
  • • Caller intent clear
  • • Type-safe enum
AI-ASSISTED WORKFLOW

CLEAN CODE
DENGAN AI TOOLS

AI coding tools bisa mempercepat refactor, tapi kamu tetap pegang kendali kualitas. Berikut tiga workflow konkret untuk clean-code tasks di sesi ini.

⚡

Cursor Composer — Rename Refactor Multi-File

CURSOR

Saat kamu perlu rename variable d → activeUsers di seluruh codebase, pakai Cursor Composer (Cmd+I) dengan prompt:

# Cursor Composer prompt
Rename variable `d` to `activeUsers` in src/services/user-service.ts
and update all downstream usages across the codebase. Keep type
signatures, update JSDoc, and run the linter after.
🤖

GitHub Copilot — Extract Function Suggestion

COPILOT

Select 15 baris di processOrder(), lalu open Copilot Chat dan minta:

# Copilot Chat prompt
/fix Extract the highlighted block into a helper function called
`calculateSubtotal(items)`. Add TypeScript types, JSDoc, and a
unit test using our existing Vitest setup.
🧠

Claude Code — Code Smell Audit

CLAUDE CODE

Di CLI root project, jalankan Claude Code interactive dan minta audit dengan prompt:

# Claude Code prompt
Audit src/services/ for the top 5 clean-code smells (long functions,
primitive obsession, god classes, magic numbers, unclear names).
For each, cite file_path:line_number, explain why it's a smell,
and suggest a concrete refactor. Don't change code yet — just report.
⚠️ Pitfall yang harus diwaspadai: AI tools kadang "over-abstract" — jangan terima saran refactor yang bikin kode lebih sulit dibaca, walaupun lebih pendek. Rule of three tetap berlaku.
QUIZ • VALIDATE

TEST
PEMAHAMAN

Kamu menemukan kode serupa di 2 tempat berbeda (masalah yang sama). Apa yang harus kamu lakukan menurut rule-of-three?

A. Langsung extract ke utility function supaya DRY
B. Biarkan duplicate. Tunggu kemunculan ketiga sebelum abstract
C. Buat interface abstrak untuk semua use case masa depan
D. Rewrite salah satu agar tidak mirip, DRY bukan prioritas

Jawabannya B — premature abstraction lebih mahal daripada duplication jangka pendek. Dua kemunculan belum cukup data untuk memilih abstraksi yang benar; tunggu kemunculan ketiga, kamu akan lebih yakin apa yang invariant dan apa yang varies.

REFACTOR

📝 Task

Refactor function berikut menjadi versi clean-code. Fokus pada naming, function decomposition, dan magic number elimination. Pakai Cursor Composer atau Claude Code untuk mempercepat.

function p(u, i) {
  let r = 0;
  for (let x = 0; x < i.length; x++) {
    if (u.t === 1) r += i[x].p * 0.9;
    else if (u.t === 2) r += i[x].p * 0.8;
    else r += i[x].p;
  }
  if (r > 1000000) r -= 50000;
  return r;
}
🤖 AI PROMPT TEMPLATE
Refactor the function `p` into clean-code form. Requirements:
1. Rename variables and function for clarity
2. Extract magic numbers to named constants
3. Replace the `u.t === 1` dispatch with a named enum
4. Split into sub-functions if needed
5. Add TypeScript types and a minimal Vitest test
