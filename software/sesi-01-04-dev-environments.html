KONSEP 1/3

EDITOR:
INVEST 1 JAM, HEMAT 100

Setup editor yang proper satu kali, rasakan return-nya setiap hari. Minimum harus punya: format-on-save, lint-on-type, multi-cursor, workspace-search cepat, dan git blame inline.

VS CODE SETTINGS.JSON (minimum)
{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": "explicit",
    "source.organizeImports": "explicit"
  },
  "files.trimTrailingWhitespace": true,
  "files.insertFinalNewline": true,
  "git.autofetch": true,
  "git.confirmSync": false,
  "terminal.integrated.defaultProfile.osx": "zsh"
}

KONSEP 2/3

DEVCONTAINERS:
CLONE & RUN

Devcontainer mendefinisikan environment development di dalam Docker image — sehingga semua developer (dan CI) punya versi Node/Python/DB/tool yang exactly sama. Bye-bye "works on my machine".

.devcontainer/devcontainer.json
{
  "name": "myapp-dev",
  "dockerComposeFile": "../compose.yml",
  "service": "app",
  "workspaceFolder": "/workspace",
  "features": {
    "ghcr.io/devcontainers/features/node:1": {
      "version": "20"
    }
  },
  "postCreateCommand": "pnpm install",
  "customizations": {
    "vscode": {
      "extensions": ["dbaeumer.vscode-eslint"]
    }
  }
}
compose.yml
services:
  app:
    build: .
    volumes:
      - ..:/workspace:cached
    environment:
      DATABASE_URL: postgres://dev:dev@db/app
    depends_on: [db, redis]
  db:
    image: postgres:16
    environment:
      POSTGRES_USER: dev
      POSTGRES_PASSWORD: dev
  redis:
    image: redis:7-alpine
📝 ONBOARDING CHECKLIST
  1. git clone <repo>
  2. Open di VS Code → "Reopen in Container"
  3. Tunggu 5 menit (build + postCreateCommand)
  4. pnpm dev → siap ngoding
KONSEP 3/3

SHELL:
10× LEBIH CEPAT

Tool CLI modern menggantikan tool lama dengan interface yang lebih baik dan performa 10× lebih cepat. Install lima ini dan rasakan bedanya dalam satu hari.

🔍

ripgrep (rg)

replaces grep

Regex search seluruh repo, respect .gitignore, parallel by default. 10-100× lebih cepat daripada grep -R.

rg "processOrder" --type ts -l
🎯

fzf

fuzzy finder

Fuzzy-find file, git branch, history, process — interactively. Ctrl+R history jadi interaktif.

# fuzzy checkout branch
git checkout $(git branch | fzf)
📁

zoxide (z)

replaces cd

Jump ke folder dengan keyword, bukan full path. z koopsud → jump ke project.

z proj   # jump to most-used dir matching "proj"
⭐

starship

prompt

Cross-shell prompt yang tampilkan git branch, package version, kube context — tanpa slow down.

🐙

lazygit / gh

git TUI + CLI

lazygit untuk staging/commit/rebase TUI; gh untuk PR/issue di terminal.

QUIZ • VALIDATE

TEST
PEMAHAMAN

Tim kamu bermigrasi ke devcontainer. Seorang developer komplain "di mesin gue bisa, di devcontainer error". Apa langkah pertama yang paling tepat?

A. Suruh dia balik ke setup lokal, devcontainer tidak worth
B. Justru ini goal-nya — reproduksi error dan fix di devcontainer config supaya semua anggota tim terkunci di state yang sama
C. Copy node_modules dari mesin lokalnya ke devcontainer
D. Install semua dependency yang missing secara manual di dalam container

Jawabannya B — tujuan devcontainer justru expose drift environment. Kalau bekerja di host tapi fail di container, hampir pasti ada implicit dependency yang belum masuk config. Fix itu sekarang, sehingga semua anggota tim (dan CI) berjalan di state identik.
