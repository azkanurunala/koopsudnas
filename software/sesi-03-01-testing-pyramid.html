KONSEP 1/3

PYRAMID VS
ICE CREAM CONE

Testing pyramid dipopulerkan Mike Cohn: banyak unit test (cepat, murah), lebih sedikit integration, sangat sedikit E2E. Ice cream cone adalah kebalikannya — anti-pattern yang bikin CI pipeline 30 menit dan full of flaky failures.

✅ HEALTHY PYRAMID
// Typical proportions
E2E           :   5%   // ~20 tests
Integration   :  20%   // ~80 tests
Unit          :  75%   // ~300 tests

// Runtime on CI
Unit          :  < 10s
Integration   :  < 60s
E2E           :  < 5 min

Feedback loop cepat. Bug ketangkep di level paling murah.

❌ ICE CREAM CONE
// Anti-pattern proportions
Manual QA     :  40%
E2E           :  30%
Integration   :  20%
Unit          :  10%

// Symptoms
- CI pipeline 30+ min
- Flaky tests retry 3x
- Bug ditemukan di staging

Feedback loop lambat. Regressions bocor ke production.

📝 RULE OF THUMB
  • • Unit: single function/class, no I/O, millisecond-speed
  • • Integration: 2+ modules atau satu module + real database, <1s per test
  • • E2E: user journey lewat UI atau API surface, few tapi critical path
KONSEP 2/3

ISOLATION &
DETERMINISM

Test yang flaky lebih berbahaya daripada test yang gagal — tim akan mulai meng-ignore. Dua akar masalah: shared state dan non-deterministic input (waktu, random, network, urutan test).

❌ FLAKY — DEPENDS ON CLOCK
test('token expires after 1 hour', () => {
  const token = createToken();
  // waits real 60 minutes — or fails when clock drifts
  setTimeout(() => {
    expect(isExpired(token)).toBe(true);
  }, 3600000);
});
✅ DETERMINISTIC — FAKE TIMER
test('token expires after 1 hour', () => {
  vi.useFakeTimers();
  const token = createToken();
  vi.advanceTimersByTime(3600 * 1000 + 1);
  expect(isExpired(token)).toBe(true);
  vi.useRealTimers();
});

Runtime: 2ms. Zero flake. Bebas urutan eksekusi.

KONSEP 3/3

MOCK, STUB,
CONTRACT TEST

Mock bukan solusi universal. Over-mocking bikin test "menguji mock" bukan code. Aturan: mock boundary eksternal (network, DB, filesystem), jangan mock collaborator internal yang sama tim-nya.

🔧

Stub

Return canned response. Contoh: db.getUser always returns sample user.

🎭

Mock

Stub + assertion atas interaksi (berapa kali dipanggil, dengan argumen apa).

🧩

Fake

Implementasi lightweight yang real. Contoh: in-memory SQLite ganti PostgreSQL.

📜

Contract Test (Pact)

Verifikasi bahwa mock/stub yang dipakai consumer masih sesuai dengan kontrak producer — di CI pada kedua sisi.

SIDE-BY-SIDE

FLAKY TEST VS
STABLE E2E

❌ FLAKY
test('login', async ({ page }) => {
  await page.goto('/login');
  await page.click('.btn');
  await page.waitForTimeout(2000);
  expect(await page.url())
    .toContain('/dashboard');
});
  • • Selector .btn ambigu
  • • Fixed sleep 2000ms
  • • Tidak ada retry strategy
  • • Tidak ada trace on failure
✅ STABLE
test('login', async ({ page }) => {
  await page.goto('/login');
  await page.getByLabel('Email')
    .fill('user@test.id');
  await page.getByRole('button',
    { name: 'Sign in' }).click();
  await expect(page)
    .toHaveURL(/\/dashboard/);
});
  • • Role-based selectors (accessible)
  • • Auto-waiting built-in
  • • Retries di playwright.config
  • • Trace on first retry
AI-ASSISTED WORKFLOW

TESTING PYRAMID
DENGAN AI TOOLS

AI tools bisa generate boilerplate test dan audit coverage gap dalam hitungan detik. Tiga workflow yang wajib kamu kuasai untuk testing tasks:

🤖

GitHub Copilot — Unit Test Skeleton

COPILOT

Select function di editor, panggil Copilot Chat dengan slash command:

# Copilot Chat prompt
/tests Generate Vitest unit tests for the selected function. Cover:
1. Happy path with typical input
2. Empty/null edge cases
3. Boundary values (0, 1, MAX_INT)
4. Error branches (throws, rejections)
Use AAA pattern (Arrange-Act-Assert) and descriptive names.
🧠

Claude Code — Coverage Gap Audit

CLAUDE CODE

Di root project, jalankan Claude Code dengan akses ke coverage report:

# Claude Code prompt
Read coverage/lcov.info and identify the top 10 critical branches
that are uncovered in src/services/payment/. For each gap, cite
file:line, explain the risk of shipping without the test, and
propose a test name + AAA skeleton. Don't write files yet.
⚡

Cursor Composer — Manual → Automated

CURSOR

Paste manual QA script dari QA tim, minta Cursor convert ke Playwright:

# Cursor Composer prompt
Convert this manual test script into a Playwright E2E test in
tests/e2e/checkout.spec.ts. Use role-based selectors, avoid
waitForTimeout, add a trace-on-retry. Manual script:
1. Open /cart
2. Add item SKU-100 to cart
3. Click "Checkout" button
4. Verify total equals Rp 150.000
⚠️ Pitfall: AI kadang generate test yang cuma "verify the mock was called" — test tanpa nilai. Selalu review: apakah test ini akan gagal kalau saya break production code?
QUIZ • VALIDATE

TEST
PEMAHAMAN

Tim kamu punya 500 E2E test, 50 integration, dan 20 unit test. CI pipeline makan 40 menit dan sering flaky. Diagnosis paling tepat?

A. Tambahkan retry 3x di E2E supaya flaky hilang
B. Upgrade CI runner ke mesin yang lebih besar
C. Ice cream cone anti-pattern — push logic ke unit test, kurangi E2E ke critical path saja
D. Jalankan E2E hanya saat release, skip di PR CI

Jawabannya C — distribusinya terbalik (ice cream cone). Retry, upgrade runner, atau skip di PR hanya menunda rasa sakit. Root-cause-nya: logic yang harusnya diuji di unit level dipaksa lewat UI. Refactor ke pyramid yang sehat.

📝 Task

Kamu dapat function calculateShipping(). Tulis unit test Vitest (3 kasus), lalu 1 integration test yang memanggilnya lewat HTTP handler, lalu 1 E2E test Playwright yang validasi di UI checkout.

function calculateShipping(weight, zone) {
  if (zone === 'domestic') return weight * 2000;
  if (zone === 'regional') return weight * 5000;
  if (zone === 'international') return weight * 15000 + 50000;
  throw new Error(`Unknown zone: ${zone}`);
}
🤖 AI PROMPT TEMPLATE
Generate a testing pyramid for calculateShipping():
1. Unit: 3 Vitest tests (happy path, boundary, error branch)
2. Integration: 1 supertest test hitting POST /api/shipping/quote
3. E2E: 1 Playwright test on /checkout that validates total
Use role-based selectors, AAA pattern, and descriptive names.
RECAP

KEY
TAKEAWAYS

🔺 Pyramid, bukan cone
75% unit, 20% integration, 5% E2E — feedback cepat, coverage luas.
🧪 Deterministic first
Fake time, fake random, fake network — flaky test adalah bug.
🎭 Mock di boundary saja
Network, DB, filesystem. Jangan mock collaborator internal.
🤖 AI untuk boilerplate
Copilot/Cursor/Claude Code generate skeleton, kamu review value.
