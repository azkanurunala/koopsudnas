KONSEP 1/3

SCIENTIFIC
DEBUGGING METHOD

Debugging bukan guesswork. Disiplin seperti scientific method: satu hipotesis per eksperimen, catat hasil, ubah satu variabel saja.

❌ SHOTGUN DEBUGGING
// Add console.log everywhere
console.log('here 1', x);
console.log('here 2', y);
console.log('wtf?', z);
// change 3 things at once
// ¯\_(ツ)_/¯ it works now?

Tidak tahu apa yang benar-benar memperbaikinya. Bug balik lagi minggu depan.

✅ SCIENTIFIC METHOD
1. Observe: "Order total wrong"
2. Reproduce: minimal case
3. Hypothesis: "discount double-
   applied when coupon + tier"
4. Experiment: isolate one var
5. Observe result
6. Confirm / revise
7. Write failing test
8. Fix + test goes green

Reproducible. Test melindungi dari regression.

📝 RUBBER DUCK DEBUGGING

Jelaskan problem ke seseorang (atau bebek karet) baris per baris. 70% bug ketemu hanya dari act of explaining — kamu dipaksa artikulasi asumsi yang selama ini implicit.

KONSEP 2/3

GIT BISECT:
BINARY SEARCH HISTORY

"Bug ini kapan mulainya?" — git bisect jawab ini dalam log₂(n) langkah. 1024 commit = hanya 10 tes manual untuk nemu commit yang memperkenalkan bug.

MANUAL BISECT
$ git bisect start
$ git bisect bad HEAD              # known broken
$ git bisect good v2.3.0           # last known good

# Git checks out midpoint commit
# You test manually or run the repro
$ git bisect good   # or
$ git bisect bad

# Repeat. Git narrows down. Usually 5-12 iterations.
# Git prints: "abc123def is the first bad commit"

$ git bisect reset
AUTOMATED BISECT (SUPER POWERFUL)
$ git bisect start HEAD v2.3.0
$ git bisect run npm test -- --filter checkout
# Git auto-runs your test on each midpoint.
# Walks away, comes back, answer ready.

# Exit code semantics:
# 0        = good
# 1-124    = bad
# 125      = skip (cannot test this commit)
KONSEP 3/3

PROFILING:
FLAME GRAPH LITERACY

Jangan optimize tanpa data. Profile dulu, baca flame graph, baru optimize hotspot yang terbukti. "Premature optimization is the root of all evil" — Knuth.

CPU Profilers

  • • perf (Linux, kernel-level)
  • • pprof (Go, also multi-lang)
  • • py-spy (Python, sampling)
  • • Chrome DevTools Performance
  • • Node --inspect

Memory Profilers

  • • Chrome heap snapshot
  • • Node --inspect + heap diff
  • • memray (Python)
  • • valgrind --massif
  • • jemalloc stats
HOW TO READ A FLAME GRAPH
  • • X-axis = frekuensi (lebar = % CPU yang dihabiskan). Bukan waktu.
  • • Y-axis = stack depth. Bawah = entry point, atas = leaf function.
  • • Warna = biasanya random (untuk kontras), bukan semantic
  • • Plateau lebar di atas = hot leaf. Itu optimization target.
  • • Plateau lebar di bawah = entry point hot. Cek caller pattern.
SIDE-BY-SIDE

GUESS VS
MEASURE

❌ "FEELS SLOW" REFACTOR
// Rewrite to "optimize"
// - Replace map with for
// - Cache intermediate results
// - Add memoization everywhere
// - 2 weeks of work

// Result: 3% faster
// Real bottleneck was N+1
// query in a different file
  • • Effort di tempat salah
  • • Kode lebih komplex
  • • Real perf problem masih ada
✅ PROFILE FIRST
1. Run profiler under load
2. Open flame graph
3. Spot: 73% time in db.query
4. Drill: called 1000x per req
5. Root cause: N+1
6. Fix: JOIN or batch
7. Re-measure: 20x faster
8. Commit + test + alert
  • • 2 jam kerja, dampak besar
  • • Data yang bicara
  • • Fix targeted

QUIZ • VALIDATE

TEST
PEMAHAMAN

Bug regression ditemukan di production. Last known good adalah 2 minggu yang lalu (≈500 commits). Apa pendekatan paling efisien untuk menemukan commit yang memperkenalkan bug?

A. Manual review satu per satu 500 commit — paling teliti
B. Tanya tim siapa yang commit terakhir — biasanya mereka tahu
C. Git bisect dengan automated repro script — log₂(500) ≈ 9 iterasi
D. Revert semua 500 commit sekaligus, lalu replay satu per satu

Jawabannya C — git bisect binary search hanya butuh ~9 iterasi untuk 500 commit. Kalau kamu punya repro script yang return exit code, git bisect run bahkan auto-menemukan commit penyebabnya. Manual review atau tanya tim tidak scale.

📝 Task

  1. git bisect start HEAD HEAD~50
  2. git bisect run npm test -- checkout
  3. Catat commit hash + diff yang bermasalah
  4. npx clinic flame -- node server.js
  5. autocannon -c 50 -d 20 http://localhost:3000/api/quote
  6. Buka flame graph di browser, identifikasi hotspot teratas
