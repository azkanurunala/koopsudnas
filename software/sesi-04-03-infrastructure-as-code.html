KONSEP 1/3

TERRAFORM:
DECLARE DESIRED STATE

Terraform mendeskripsikan infrastruktur yang kamu inginkan, bukan langkah-langkah untuk membuatnya. Cycle: plan → review → apply.

terraform {
  required_providers {
    aws = { source = "hashicorp/aws", version = "~> 5.0" }
  }
  backend "s3" {
    bucket         = "tf-state-prod"
    key            = "api/terraform.tfstate"
    region         = "us-east-1"
    dynamodb_table = "tf-locks"
  }
}

resource "aws_s3_bucket" "uploads" {
  bucket = "my-app-uploads-prod"
  tags   = { Env = "prod", ManagedBy = "terraform" }
}
KONSEP 2/3

MODULES:
REUSABLE COMPOSITION

Module = Terraform config yang bisa di-pakai ulang dengan inputs. Satu module definiskan "service", di-invoke per environment dengan values berbeda.

# modules/api-service/main.tf
variable "name"    { type = string }
variable "env"     { type = string }
variable "replicas" { type = number, default = 3 }

resource "aws_ecs_service" "this" {
  name            = "${var.name}-${var.env}"
  desired_count   = var.replicas
  # ... more
}

# envs/prod/main.tf
module "api" {
  source   = "../../modules/api-service"
  name     = "api"
  env      = "prod"
  replicas = 10
}
KONSEP 3/3

GITOPS:
GIT = SOURCE OF TRUTH

Dengan GitOps (ArgoCD, Flux), cluster melakukan pull desired state dari git repo. Merge PR = deploy. Drift (cluster state ≠ git) terdeteksi otomatis dan di-reconcile.

# ArgoCD Application manifest
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: api-prod
spec:
  source:
    repoURL: https://github.com/org/infra.git
    path: apps/api/envs/prod
    targetRevision: main
  destination:
    server: https://kubernetes.default.svc
    namespace: api-prod
  syncPolicy:
    automated: { prune: true, selfHeal: true }
WORKFLOW

PLAN → REVIEW →
APPLY CYCLE

1️⃣

Write + terraform plan

Edit .tf files. Plan menampilkan preview diff (+/−/~).

2️⃣

PR + Review

CI run plan, posting diff di PR. Reviewer check blast radius.

3️⃣

Merge + Apply

Merge → CI apply (atau ArgoCD sync). State update ke remote.

QUIZ

TEST
PEMAHAMAN

Kamu terraform apply dari laptop tanpa remote backend, lalu teammate apply dari laptop-nya juga. Apa yang terjadi?

A. Terraform detect conflict dan abort otomatis
B. State file overwrite — resource drift, potentially destroy data
C. Git merge conflict pada file .tf
D. Cloud provider reject kedua apply karena rate limit

Jawaban B — local state file dua laptop tidak ter-sinkron. Kedua apply bisa overwrite state, menghasilkan resource drift atau destroy. Selalu pakai remote backend (S3 + DynamoDB lock) untuk team workflow.
