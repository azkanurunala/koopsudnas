KONSEP 1/3

TOOL
TAXONOMY

Tool AI coding 2026 dibagi jadi tiga kategori utama berdasarkan tempat AI bekerja. Memahami kategori ini adalah langkah pertama memilih tool yang tepat.

IDE-NATIVE

Di dalam editor

  • • GitHub Copilot
  • • Cursor
  • • Windsurf
  • • JetBrains AI

Inline completion, chat side-panel, multi-file edit via Composer/Cascade.

CLI AGENT

Di terminal

  • • Claude Code
  • • Aider
  • • OpenAI Codex CLI
  • • Gemini CLI

Autonomous agent, read-plan-edit loop, long-running tasks, Git-native workflow.

UI-FIRST GEN

Di browser

  • • v0.dev
  • • Bolt.new
  • • Lovable
  • • Replit Agent

Prompt → full component/app, preview instan, code export untuk handoff.

💡 INSIGHT

Tidak ada "winner". Tim modern pakai kombinasi: Cursor untuk daily coding, Claude Code untuk refactor besar, v0 untuk prototype UI. Pelajari satu per satu lalu mix.

KONSEP 2/3

POSITIONING
MATRIX

Sumbu X: autonomy (suggestion → autonomous agent). Sumbu Y: scope (single file → full codebase). 

# Mental model 2x2

                FULL CODEBASE SCOPE
                         ▲
                         │
     Claude Code ────────┼──────── Bolt.new
     Cursor Composer     │         Replit Agent
                         │
  ───────────────────────┼───────────────────────▶ AUTONOMY
     Copilot Inline      │         v0.dev
     Aider (chat)        │         Windsurf Cascade
                         │
                         ▼
                  SINGLE FILE SCOPE

# Left side  = assistive (you drive)
# Right side = autonomous (AI drives)
# Top        = multi-file / whole repo awareness
# Bottom     = line/function level suggestions
HIGH AUTONOMY + FULL SCOPE

Claude Code, Bolt, Replit Agent — pakai saat task jelas dan kamu siap review besar.

LOW AUTONOMY + SINGLE FILE

Copilot inline, Aider chat — pakai saat kamu masih eksplorasi dan butuh fine control.

KONSEP 3/3

SELECTION
CRITERIA

Lima faktor untuk memilih. Jawab jujur, lalu shortlist tool yang cocok.

📏

Codebase Size

< 10k LOC: Copilot / v0 cukup. 10k–100k: Cursor + indexing. > 100k: Claude Code + MCP + retrieval.

🔁

Workflow Style

Keyboard-centric / terminal person → Claude Code, Aider. GUI-loving → Cursor, Windsurf. Design-first → v0, Bolt.

👥

Team Adoption

Tim yang sudah pakai VSCode → Copilot/Cursor gesekan rendah. Tim polyglot → CLI agent netral.

💰

Budget & Licensing

Copilot $10/mo per user termurah. Cursor Pro $20. Claude Code pay-per-token (variable). Aider bring-your-own-key (flexible).

🔒

Privacy & Compliance

Copilot Enterprise punya zero-data-retention. Cursor Privacy Mode. Untuk regulated industry pilih yang punya BAA / SOC2.

AI-ASSISTED WORKFLOW

"FIND A BUG"
CROSS-TOOL COMPARISON

Satu task identik — "temukan bug race condition di order processing" — dijalankan di tiga tool berbeda. Lihat perbedaan pendekatan, prompt, dan output.

💬

Copilot Chat — Single File Focus

COPILOT

Buka file yang dicurigai, invoke Copilot Chat dengan slash command:

# Copilot Chat prompt (file context)
/explain #file:order-processor.ts
Are there any race conditions in how we handle concurrent order
submissions? Focus on the processOrder() function. If yes, cite
line numbers and propose a fix using a mutex or atomic DB update.
🎼

Cursor Composer — Codebase Wide

CURSOR

Open Composer (Cmd+I), attach codebase, ask the wide question:

# Cursor Composer prompt
@codebase Trace all paths that write to the `orders` table.
Identify any that don't use a transaction or row lock. List each
as file:line with the risk level (HIGH/MED/LOW). Don't fix yet —
just report so I can review.
🧠

Claude Code — Autonomous Plan Mode

CLAUDE CODE

Di CLI, jalankan plan mode untuk investigasi mendalam:

# Claude Code (plan mode)
/plan Investigate potential race conditions in order processing.
Steps: (1) grep for order writes, (2) map transaction boundaries,
(3) simulate concurrent runs, (4) propose fix with test cases.
Report as markdown with file_path:line_number citations.
🔑 Kunci perbedaan: Copilot fokus 1 file (cepat, terbatas). Cursor Composer scope codebase (akurat, perlu indexing). Claude Code autonomous plan (mendalam, butuh review lebih besar). Pilih berdasarkan skala task.
QUIZ • VALIDATE

TEST
PEMAHAMAN

Tim kamu maintain monorepo 250k LOC, butuh refactor cross-service besar, developer mostly pakai neovim + terminal. Tool AI coding mana yang paling tepat sebagai primary tool?

A. v0.dev karena paling cepat generate UI
B. GitHub Copilot inline di VSCode karena termurah
C. Claude Code CLI — cocok terminal workflow, autonomous plan mode untuk refactor besar, long-context untuk repo 250k LOC
D. Bolt.new karena bisa generate full project

Jawabannya C — Claude Code CLI menang di tiga kriteria krusial: (1) sesuai workflow terminal/neovim, (2) autonomous plan mode + skills untuk refactor besar, (3) long-context window + MCP untuk codebase 250k LOC. Copilot/v0/Bolt tidak pas untuk skala & workflow ini.

RECAP

KEY
TAKEAWAYS

🗺️ Three categories
IDE-native, CLI agent, UI-first gen — masing-masing punya sweet spot.
🎯 Scope × Autonomy
Dua sumbu utama untuk menempatkan setiap tool di mental map.
📊 Five criteria
Size, workflow, team, budget, privacy — jawab sebelum pilih.
🤖 Mix, don't pick one
Tim modern kombinasi 2-3 tool. Kuasai dulu, baru mix.
