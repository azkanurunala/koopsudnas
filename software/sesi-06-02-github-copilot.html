KONSEP 1/3

AUTOCOMPLETE:
PROMPT-IN-COMMENTS

Copilot inline completion membaca konteks sekitar cursor — comment, nama variabel, imports, function signature. Gunakan comment sebagai prompt untuk mendapatkan suggestion yang akurat.

❌ VAGUE CONTEXT
// sort
function sortItems(items) {
  // cursor here
}

Copilot tidak tahu: sort by apa? ascending? tipe items apa?

✅ PROMPT-IN-COMMENT
// Sort products by createdAt DESC,
// tie-break by priority DESC.
// Return new array, don't mutate.
function sortProducts(products: Product[]) {
  // cursor here — Copilot knows exactly what to do
}

Intent + sort keys + immutability + tipe. Suggestion langsung akurat.

📝 AUTOCOMPLETE TIPS
  • • File name matters: user-repository.ts biases Copilot to DB patterns
  • • Imports first: import libs yang kamu mau (e.g. zod) — Copilot akan pakai
  • • Accept partial: Tab untuk full, Ctrl+→ untuk per-word, esc untuk reject
  • • Cycle suggestions: Alt+] untuk lihat alternative
KONSEP 2/3

COPILOT CHAT
& SLASH COMMANDS

Chat panel menjadi "second brain". Gunakan slash commands untuk task umum, #file: dan @workspace untuk scope context dengan presisi.

CORE SLASH COMMANDS
# Penjelasan
/explain          -- jelaskan selected code / file saat ini
/fix              -- perbaiki bug/warning di selection
/tests            -- generate unit tests untuk selection
/doc              -- tambah JSDoc/docstring
/optimize         -- perbaiki performance
/new              -- scaffold project/file baru

# Scope references
#file:path/to/x.ts    -- include file sebagai context
#selection            -- gunakan text yang di-highlight
@workspace            -- scope ke seluruh workspace
@terminal             -- interpret output terminal
DAILY BREAD

/tests setelah tulis function, /fix untuk error, /explain untuk legacy code.

POWER USER

@workspace untuk pertanyaan arsitektur, #file: chain untuk cross-file refactor.

KONSEP 3/3

COPILOT WORKSPACE
& AGENTS

Copilot Workspace = task-to-PR flow. Kamu kasih task (bisa dari issue GitHub), Copilot generate spec → plan → code → test → PR draft. Kamu review setiap tahap.

WORKSPACE FLOW
1. TASK     → "Add dark mode toggle to settings page"
2. SPEC     → Copilot drafts: user stories, acceptance criteria
3. PLAN     → file-by-file changes proposed (you edit)
4. CODE     → implementation per file (you review)
5. TEST     → tests auto-generated
6. PR       → draft PR created, you polish & merge
🎯

Best For

Well-defined feature, bug dari issue tracker, boilerplate scaffolding.

⚠️

Not For

Task exploratory, refactor arsitektur, debugging kompleks yang butuh interaksi cepat.

🧠

Human Role

Review spec, revise plan, verify tests — kamu tetap decision-maker.

LIVE DEMO

FROM ZERO
TO TESTED FEATURE

STEP-BY-STEP
# 1. Prompt-in-comment untuk scaffold
// Create an Express route POST /api/orders
// - Validate body with zod schema (items, email)
// - Call orderService.create()
// - Return 201 with order JSON, 400 on invalid

# 2. Copilot generates handler — accept with Tab

# 3. Select handler, open chat, /tests
/tests Generate Vitest tests covering happy path, invalid body,
and service failure. Use supertest for integration.

# 4. Run tests, fix with /fix
/fix The "invalid email" test is failing — the zod schema needs
a proper email validator.

# 5. Ask for docs
/doc Add JSDoc with @param, @returns, @throws
✅ WHAT WORKED

Comment spec → accurate handler in 1 tab. Tests generated cover edge cases. Flow ~5 menit untuk feature yang biasanya 30 menit.

⚠️ WHAT TO WATCH

Copilot kadang invent import yang tidak ada. Selalu check imports, lint sebelum commit, run tests.

Custom Instructions (.github/copilot-instructions.md)

TEAM CONVENTIONS

Inject team conventions otomatis ke setiap prompt Copilot:

# .github/copilot-instructions.md
# Project Conventions

- Language: TypeScript strict mode, no `any`
- Validation: zod for all external input
- Testing: Vitest, file-per-module, `describe` block per function
- Errors: Custom error classes extending AppError (see src/errors.ts)
- Imports: absolute paths from `@/` alias
- Database: Drizzle ORM, transactions for multi-write operations

Always follow these when suggesting code.
🔑 Pro tip: Custom instructions adalah force-multiplier. Sekali setup, setiap Copilot suggestion otomatis "paham" team conventions tanpa kamu ulang-ulang jelaskan.
QUIZ • VALIDATE

TEST
PEMAHAMAN

Tim kamu ingin memastikan Copilot selalu suggest code yang pakai zod untuk validasi dan Vitest untuk test — tanpa harus ingatkan di setiap chat. Cara terbaik?

A. Tulis comment "gunakan zod dan vitest" di setiap file
B. Buat .github/copilot-instructions.md yang mencantumkan konvensi tim
C. Ganti ke Cursor karena support .cursorrules
D. Selalu awali chat prompt dengan "use zod and vitest"

Jawabannya B — .github/copilot-instructions.md di-read otomatis oleh Copilot Chat dan di-inject ke setiap prompt. Satu kali setup, seluruh tim dapat manfaat. Opsi A dan D bisa dilupakan; opsi C mengganti tool bukan solusinya.
