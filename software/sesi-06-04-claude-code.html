KONSEP 1/3

CLI BASICS
& SLASH COMMANDS

Claude Code adalah CLI interactive — kamu bisa chat, beri task, atau ketik slash command. Semua state tersimpan dalam session; keluar dengan /exit.

BASIC WORKFLOW
# Install
$ npm install -g @anthropic-ai/claude-code

# Start interactive session in project root
$ cd my-project
$ claude

# Inside session — natural language task
> Refactor src/auth/login.ts to use the new session API

# Or slash commands
> /plan Add rate limiting to all public endpoints
> /commit                 # stage + commit with generated msg
> /review                 # review current diff
> /clear                  # reset context window
> /agents                 # list & invoke subagents
> /hooks                  # manage hooks
> /mcp                    # manage MCP servers
> /help                   # all commands
📁 CLAUDE.md

File di root yang otomatis di-load sebagai project memory — konvensi, arsitektur, do/don't.

🎯 CONTEXT

Claude Code bisa read file, run bash, grep, edit. Setiap aksi minta permission (bisa di-auto-allow via settings).

# Claude reads files, builds a plan:
# Plan: Migrate to session cookies
# 1. Install `iron-session`
# 2. Create src/lib/session.ts with wrapper
# 3. Update login handler, logout handler
# 4. Replace withAuth HOC to read from session
# 5. Remove JWT code after migration verified
# 6. Update tests

# You review, approve, then Claude executes step-by-step
SKILLS

Reusable domain expertise

# Skills are markdown files that teach Claude a workflow.
# Example: ~/.claude/skills/review-pr/SKILL.md

---
name: review-pr
description: Review GitHub PR for security, performance, style
---

When invoked, fetch the PR diff, analyze for:
1. SQL injection, XSS, auth bypass
2. N+1 queries, unbounded loops
3. Missing error handling
4. Style violations from CLAUDE.md

Report as markdown with severity levels.
HOOKS

Automation on tool events

# ~/.claude/settings.json
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Edit|Write",
      "hooks": [{
        "type": "command",
        "command": "npx prettier --write \"$FILE\" && npm run lint:fix"
      }]
    }],
    "Stop": [{
      "hooks": [{ "type": "command", "command": "npm test" }]
    }]
  }
}
KONSEP 3/3

LONG CONTEXT
+ MCP

Claude Code support context window 200k–1M token. Untuk source di luar file system (database, API, Jira, GitHub), pakai MCP servers — standardized tool integration.

ENABLE MCP SERVERS
# ~/.claude/settings.json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/projects"]
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": { "GITHUB_TOKEN": "${GITHUB_TOKEN}" }
    },
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres",
               "postgresql://localhost/dev"]
    }
  }
}

PLAN MODE
REAL EXAMPLE

TASK: ADD MULTI-TENANCY
> /plan I need to add multi-tenancy to this app. Every user belongs
> to one organization. All queries must filter by organization_id.
> Plan the migration carefully — this is a big change.

# Claude reads relevant files (db schema, services, middleware)
# Outputs structured plan:

# Multi-tenancy Migration Plan

## Phase 1: Schema (1-2 hours)
- [ ] Add `organizations` table
- [ ] Add `organization_id` FK to: users, projects, items, logs
- [ ] Create migration 20260415_multi_tenant.sql
- [ ] Backfill: every existing user → default org

## Phase 2: Auth middleware (30 min)
- [ ] Inject `req.organizationId` from user session
- [ ] Create `requireOrg` middleware

## Phase 3: Query layer (2-3 hours)
- [ ] Audit all queries in src/db/queries/*
- [ ] Add organizationId filter to all reads
- [ ] Add transaction wrapper that asserts org match on writes

## Phase 4: Tests (1 hour)
- [ ] Update fixtures with orgId
- [ ] Add cross-tenant leakage test

## Risks
- Forgotten queries leak data → grep audit + tests
- Backfill rollback plan needed
- API clients may break — deprecation notice

Proceed? [y/n]
> y

# Claude executes phase by phase, asking confirmation between phases
✅ WHY IT WORKS

Plan explicit, risk-aware, phased. Kamu review sebelum execute. Tidak ada surprise.

⚠️ STILL NEED YOU

Review diff phase-by-phase. Verify data backfill. Smoke test production replica.

Subagent Orchestration

AGENTS

Delegate ke subagent spesialis untuk task panjang:

# Claude Code prompt
Spawn subagents in parallel to audit our API surface:
- security-auditor: check every public route for auth bypass,
  injection, IDOR. Report severity.
- performance-auditor: find N+1 queries, missing indexes,
  unbounded loops.
- docs-auditor: check which routes are missing OpenAPI docs.

Each subagent should write a report to reports/<agent>.md.
When all done, summarize top 10 findings across all three.
🪝

Hook-Based Automation

HOOKS

Enforce quality gates otomatis setiap Claude edit file:

# ~/.claude/settings.json — auto quality gate
{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Edit|Write",
      "hooks": [{
        "type": "command",
        "command": "node scripts/check-file-allowed.js \"$FILE\""
      }]
    }],
    "PostToolUse": [{
      "matcher": "Edit|Write",
      "hooks": [
        { "type": "command", "command": "prettier --write \"$FILE\"" },
        { "type": "command", "command": "eslint --fix \"$FILE\"" },
        { "type": "command", "command": "npm run typecheck -- --noEmit" }
      ]
    }]
  }
}
🔍

Codebase Audit via Skill

SKILLS

Skill reusable untuk audit sistematis yang bisa dipanggil kapan saja:

# ~/.claude/skills/codebase-audit/SKILL.md
---
name: codebase-audit
description: Full-repo audit for code smells, security, perf
---

Steps:
1. Read CLAUDE.md to understand project conventions
2. Scan src/ for:
   - Functions > 50 lines
   - Files > 500 lines
   - TODO/FIXME/HACK markers
   - `any` types in TypeScript
   - `console.log` in non-dev code
   - Missing error handling on async calls
3. Run `npm audit` and aggregate findings
4. Report as markdown with file:line citations
5. Suggest top 5 refactors ranked by impact

Usage: > Run the codebase-audit skill
🔑 Kombinasi killer: Plan mode untuk task besar + subagents untuk parallelism + hooks untuk quality gate + skills untuk reusability. Claude Code jadi "junior developer yang 24/7 standby".
QUIZ • VALIDATE

TEST
PEMAHAMAN

Kamu ingin memastikan setiap kali Claude Code edit file TypeScript, prettier + eslint --fix + typecheck berjalan otomatis. Fitur mana yang paling tepat?

A. Tulis di CLAUDE.md "please run prettier after every edit"
B. Konfigurasi PostToolUse hook di settings.json dengan matcher "Edit|Write"
C. Buat skill khusus "format-after-edit"
D. Manual jalankan npm run lint setelah session selesai

Jawabannya B — PostToolUse hook adalah mekanisme deterministik yang dieksekusi harness (bukan model). CLAUDE.md/Skill bisa di-ignore model; hanya hook yang guaranteed jalan. Ini cara yang benar untuk quality gate otomatis.

BUILD YOUR
FIRST SKILL

📝 Task

Buat skill commit-smart yang: (1) stage hanya file yang relevant, (2) generate conventional commit message, (3) tolak jika test failing.

# Steps

1. Install claude code: npm install -g @anthropic-ai/claude-code
2. Buat direktori: mkdir -p ~/.claude/skills/commit-smart
3. Tulis SKILL.md (template di bawah)
4. Di project, buka session: claude
5. Test: > Run the commit-smart skill
6. Iterate sampai behavior sesuai
🤖 SKILL TEMPLATE
---
name: commit-smart
description: Stage, commit with conventional message, block on test fail
---

When invoked:
1. Run `npm test` — if any test fails, STOP and report failures
2. Run `git status` and show untracked/modified files
3. Ask user which files to stage (default: all modified, no new)
4. Run `git diff --staged` to understand changes
5. Draft a conventional commit message:
   - type: feat|fix|refactor|docs|test|chore
   - scope: inferred from directory
   - subject: imperative, < 60 chars
   - body: bullet points of what/why if non-trivial
6. Show message, ask for confirmation
7. On confirm: `git commit -m "..."`
8. Report commit SHA
