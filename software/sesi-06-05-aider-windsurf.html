KONSEP 1/3

AIDER:
GIT-NATIVE AI PAIR

Aider adalah terminal AI pair programmer yang auto-commit setiap edit. Setiap perubahan jadi git commit terpisah dengan message yang descriptive — rollback gampang, history bersih.

# Install & run
pip install aider-chat
export ANTHROPIC_API_KEY=sk-ant-...

# Start with specific files in context
aider src/handlers/payment.ts src/db/orders.ts

# In aider chat:
> Extract the discount calculation into a separate function
>   called calculateDiscount. Add unit tests.

# Aider applies edits + creates commit:
# "refactor: extract calculateDiscount with unit tests"
KONSEP 2/3

WINDSURF:
CASCADE AGENT IDE

Windsurf (by Codeium) adalah IDE AI-first mirip Cursor, dengan fokus pada Cascade — agent yang dapat run long-horizon tasks dengan minimum supervision (plan → execute → self-verify).

Write Mode

Interactive pair programming seperti Cursor Composer — diff review tiap edit.

Cascade (Flow) Mode

Agent mode — Windsurf plans, executes multi-file changes, runs tests, reports back.

POWER PATTERNS

3 WORKFLOW
YANG WORTH IT

Berikut 3 pola spesifik Aider + Windsurf yang kebanyakan user baru tidak tahu.

📟

Aider — Commit-per-Edit Refactor

AIDER

Aider auto-commit setelah setiap accepted edit. Pairs dengan small-step refactoring = history yang sangat reviewable.

# Aider workflow
aider src/legacy-service.js \
  --model claude-sonnet-4-6 \
  --auto-commits

# Then give small, atomic instructions:
> Step 1: Add JSDoc types to the exports
> Step 2: Extract helper functions into a utils/ file
> Step 3: Add vitest unit test for main export

# Result: 3 clean commits, easy to review/revert
🌊

Windsurf — Cascade Long-Horizon Task

WINDSURF

Cascade mode dapat run multi-step tasks dengan self-verification — cocok untuk migrations atau framework upgrades.

# Windsurf Cascade prompt
Migrate this project from Express 4 to Hono:
1. Read existing routes under src/routes/
2. Create equivalent Hono routes
3. Update server.ts entry point
4. Run test suite, fix any failures
5. Update package.json dependencies
Report which routes still need manual review.
🔀

Hybrid — Aider for Review, Windsurf for Feature

BOTH

Pakai Windsurf untuk feature development, lalu Aider untuk small touch-ups dengan commit-clean history.

# 1. Windsurf implements feature
#    (big Cascade task, 1 messy commit)

# 2. git reset --soft HEAD~1
# 3. aider for clean-up commits:
aider src/features/new/ --auto-commits
> Split this into logical commits:
>   1. schema changes
>   2. service layer
>   3. route handlers
>   4. tests
💡 Tip: Aider hormat .aiderignore file seperti .gitignore — pakai untuk exclude secrets, large binary files, dan generated code dari context.

TEST
PEMAHAMAN

Kamu butuh reviewable git history untuk compliance — setiap perubahan harus individual commit dengan descriptive message. Tool mana paling natural fit?

A. Aider — auto-commit per edit, descriptive message by default
B. Cursor Composer — faster edits
C. Windsurf Cascade — agent mode
D. Copilot chat — inline editing

Jawaban A — Aider dirancang git-native dengan auto-commit per edit. Tools lain bisa juga (manual), tapi Aider default behavior ini membuatnya paling natural untuk compliance audit trail.
