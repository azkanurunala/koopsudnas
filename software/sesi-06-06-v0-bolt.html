KONSEP 1/3

V0:
COMPONENT-FIRST

v0.dev generates individual React components pakai shadcn/ui + Tailwind. Iterasi: chat, preview, refine, export ke project kamu via CLI.

# Workflow
1. Buka v0.dev
2. Prompt: "Pricing table with 3 tiers,
   highlighted middle, monthly/yearly toggle"
3. v0 generate component preview
4. Iterate: "make the CTA buttons larger,
   add a 'Popular' badge on middle tier"
5. Click "Copy code" or run:
   npx v0@latest add <component-id>
6. Component masuk ke components/ui/ di project
KONSEP 2/3

BOLT:
FULL-STACK PROJECT

Bolt.new generates entire project — Next.js / Vite / Astro, dengan file structure lengkap, run in-browser (WebContainers), dan 1-click deploy ke Netlify.

# Bolt prompt example
Build a todo app with:
- Next.js 15 App Router + TypeScript
- SQLite via Prisma for persistence
- Tailwind + shadcn for UI
- Server actions for CRUD
- Drag-to-reorder with dnd-kit

# Bolt generates ~20 files, runs dev server,
# shows live preview in right panel.
# Can iterate: "add dark mode toggle"
KONSEP 3/3

PRODUCTION
READINESS GAPS

UI-first tools hebat untuk prototyping, tapi ada gaps yang harus kamu isi manual sebelum ship ke produksi:

⚠️ Gaps Umum

  • • Accessibility (aria labels, keyboard nav)
  • • Error boundaries & loading states
  • • Form validation yang serious
  • • Auth & authorization
  • • Input sanitization (XSS)
  • • Performance (bundle size, SSR)
  • • SEO metadata
  • • Rate limiting

✅ Yang Benar-Benar Cepat

  • • Landing page draft
  • • Dashboard wireframe → clickable
  • • Form UI layouts
  • • Design system exploration
  • • Internal tool prototipe
  • • Stakeholder demo
  • • Hackathon MVP
COMPARISON

V0 VS
BOLT

v0.dev

Scope: Single component (mostly)

Stack: React + shadcn/ui + Tailwind

Output: Copy-paste or CLI add

Best for: Design system growth, isolated components

Pricing: Free tier + paid Vercel plans

Bolt.new

Scope: Full project

Stack: Any framework (Next, Vite, Astro, etc.)

Output: Running project in browser

Best for: MVP, hackathon, complete prototypes

Pricing: Free tier + StackBlitz paid plans

YANG WORTH IT

Cara memaksimalkan v0 + Bolt dalam professional workflow, bukan sekadar "generate and hope".

🎨

v0 — Component Prototyping

V0

Pakai v0 untuk rapid iteration komponen, lalu polish di Cursor.

# v0 prompt
Create a data table component with:
- 7 columns (configurable)
- Sort by column, multi-select rows
- Pagination (client-side)
- Search filter
- shadcn/ui styling
Then: export via "npx v0 add" ke project kamu
💡

Bolt — Design Exploration

BOLT

Sebelum commit ke framework choice, eksplor 2-3 stack alternatif pakai Bolt.

# Bolt prompt A
Build a real-time chat app:
- Next.js 15 App Router + Server Actions
- SQLite + Drizzle
- Tailwind

# Bolt prompt B (new session)
Same requirements but:
- Astro + React island
- Turso (libSQL) + Drizzle
Compare DX, bundle size, deploy complexity.
🔁

Gap-Handoff — Bolt → Cursor → Prod

HYBRID

Bolt generates, Cursor hardens. Pattern untuk ship Bolt output ke production.

# 1. Bolt generate MVP, export to GitHub
# 2. Clone locally, open in Cursor
# 3. Cursor Composer prompt:
Audit this project for production-readiness:
- Missing error boundaries
- Missing form validation (use zod)
- Missing auth middleware
- Missing test setup (add vitest)
- Missing env schema validation
Prioritize by security impact, then UX.
⚠️ Pitfall: v0/Bolt sering pakai library versi terbaru (baru release). Lock version di package.json setelah export untuk reproducibility.
QUIZ

TEST
PEMAHAMAN

Stakeholder minta mockup dashboard yang clickable dalam 1 jam untuk meeting. Kamu punya React + Tailwind familiarity. Approach terbaik?

A. Coding dari scratch di local — full control
B. Bolt.new — prompt untuk full dashboard, iterasi live
C. Figma mockup — avoid code entirely
D. Cursor — scaffold project manual

Jawaban B — Bolt paling cepat untuk use case ini (clickable dashboard dalam 1 jam). Full control (A) terlalu lama; Figma (C) tidak clickable dalam React terms; Cursor scaffold (D) lebih cocok untuk production code tapi lambat untuk demo.
