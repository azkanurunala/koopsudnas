KONSEP 1/3

MCP
ARCHITECTURE

MCP = JSON-RPC 2.0 protocol antara Host (Claude Code, Claude Desktop), Client (the agent instance), dan Server (a process exposing tools). Transport: stdio (local) atau HTTP (remote).

┌──────────┐      ┌──────────┐      ┌──────────┐
│   Host   │      │  Client  │      │  Server  │
│ (Claude  │──────│ (Agent   │──────│ (Process │
│  Code)   │      │ session) │      │ exposing │
└──────────┘      └──────────┘      │  tools)  │
                                    └──────────┘

Communication: JSON-RPC 2.0
Transport:     stdio (local) or HTTP (remote)

Server exposes:
- tools      (callable functions)
- resources  (readable data URIs)
- prompts    (templated workflows)
KONSEP 2/3

EXISTING
SERVERS

Ecosystem MCP sudah punya banyak server built-in. Cukup add ke config, langsung bisa dipakai Claude Code / Claude Desktop.

// .mcp.json (project-level) or ~/.config/claude/mcp.json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["@modelcontextprotocol/server-filesystem", "./"]
    },
    "postgres": {
      "command": "npx",
      "args": ["@modelcontextprotocol/server-postgres",
               "postgresql://localhost/dev"]
    },
    "github": {
      "command": "npx",
      "args": ["@modelcontextprotocol/server-github"],
      "env": { "GITHUB_TOKEN": "ghp_..." }
    }
  }
}
KONSEP 3/3

BUILD YOUR
OWN SERVER

Minimal MCP server pakai TypeScript SDK — deklarasikan tools, handle request, pass ke agent. Contoh: server untuk query internal analytics API.

import { Server } from "@modelcontextprotocol/sdk/server";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio";

const server = new Server({ name: "analytics", version: "1.0" });

server.setRequestHandler("tools/list", async () => ({
  tools: [{
    name: "query_events",
    description: "Query analytics events by date range",
    inputSchema: {
      type: "object",
      properties: {
        start: { type: "string" },
        end:   { type: "string" }
      }
    }
  }]
}));

server.setRequestHandler("tools/call", async (req) => {
  if (req.params.name === "query_events") {
    const data = await queryDb(req.params.arguments);
    return { content: [{ type: "text", text: JSON.stringify(data) }] };
  }
});

await server.connect(new StdioServerTransport());
SECURITY

TRUST &
PERMISSION MODEL

✅ Do

  • • Run MCP server sebagai separate process (isolation)
  • • Whitelist allowed tools di host config
  • • Log semua tool calls (audit trail)
  • • Validate inputs di server side (jangan trust agent)
  • • Scope secrets ke env var, bukan hard-code
  • • Rate limit per tool untuk prevent runaway loops

❌ Don't

  • • Kasih MCP server akses prod DB tanpa read-only role
  • • Expose exec_shell tool tanpa allow-list
  • • Share personal tokens via shared .mcp.json
  • • Skip permission prompt di host
  • • Trust tool output untuk security decisions
  • • Install random MCP servers dari internet tanpa review

Security — Read-Only Role + Audit

SECURITY

Wrap database access dalam read-only DB user, log setiap tool call, rate limit.

# In your MCP server
const pool = new Pool({
  connectionString: process.env.DB_READONLY_URL // readonly role
});

server.setRequestHandler("tools/call", async (req) => {
  audit.log({ tool: req.params.name, args: req.params.arguments });
  await rateLimiter.check(req.params.name);
  // ... execute ...
});
⚠️ Pitfall: MCP servers run in your local process space. Jangan pernah install random MCP package tanpa review source — mereka punya akses file system yang sama dengan terminal kamu.
QUIZ

TEST
PEMAHAMAN

Tim kamu mau Claude Code bisa query production Postgres untuk debug, tanpa developer perlu paste data manual. Approach paling aman?

A. Install @modelcontextprotocol/server-postgres dengan prod write credentials
B. MCP server custom dengan read-only DB role, audit logging, rate limit
C. Copy-paste query results ke Claude Code manually
D. SSH tunnel untuk Claude agent connect langsung ke DB server

Jawaban B — read-only role membatasi blast radius (tidak bisa DROP/DELETE), audit log memberi traceability, rate limit prevent runaway loops. Write credentials ke prod (A) = risk besar.
