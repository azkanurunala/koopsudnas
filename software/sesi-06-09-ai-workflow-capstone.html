SCENARIO

CAPSTONE
BRIEF

📦 Feature: "Task Reminders"

Tambahkan fitur scheduled reminders ke existing task manager app. User set reminder pada task, sistem kirim email 15 menit sebelum due time.

REQUIREMENTS
  • • API endpoint: POST /tasks/:id/reminders (time, notification_method)
  • • Background job: scheduler yang run setiap 1 menit, kirim email untuk reminder yang due
  • • UI: settings modal untuk set reminder pada task card
  • • Tests: unit (business logic), integration (scheduler), E2E (UI flow)
  • • Observability: metric reminders_sent_total, error budget tracking
PHASE 1/3

CLAUDE CODE:
PLAN + DESIGN

Mulai dengan Claude Code dalam plan mode — AI-assisted architect yang analyze codebase, surface constraints, dan draft implementation plan.

STEP 1.1 — Exploration
claude-code --plan
> Explore the current task manager codebase.
>   Identify: API structure, DB schema, background job
>   infrastructure, email sending, test patterns.
>   Report what exists & what needs to be built.
STEP 1.2 — Design Doc
> Based on exploration, write docs/design/reminders.md:
>   - API contract (OpenAPI spec)
>   - DB migration (add reminders table)
>   - Scheduler architecture (cron vs queue)
>   - Email template spec
>   - Test plan
>   - Rollout plan with feature flag
STEP 1.3 — Exit plan mode

Claude Code ExitPlanMode dengan plan yang sudah disetujui kamu. Plan file jadi reference untuk tools lain di phase selanjutnya.

PHASE 2/3

CURSOR +
COPILOT: BUILD

Handoff ke Cursor untuk multi-file scaffold, Copilot untuk inline completion pada business logic yang well-scoped.

STEP 2.1 — Cursor: Scaffold
# Cursor Composer (Cmd+I)
Read docs/design/reminders.md and scaffold:
- Prisma migration for reminders table
- src/services/reminders/ (create, list, delete)
- src/routes/reminders.ts (API handlers)
- src/jobs/reminder-scheduler.ts (cron job)
- src/services/reminders/__tests__/*.test.ts
Follow existing patterns from src/services/tasks/.
STEP 2.2 — Copilot: Fill business logic

Buka file demi file, Copilot completion untuk fill function body dengan context dari comment:

// Send email reminders for all due reminders in the last 2 minutes.
// Idempotent: use sent_at to skip already-sent reminders.
async function processDueReminders() {
  // Copilot: generates the implementation
}
PHASE 3/3

BOLT +
CLAUDE CODE: SHIP

Bolt untuk prototyping UI modal, Claude Code untuk integration + review + PR preparation.

STEP 3.1 — Bolt: UI mockup
# bolt.new
Build a reminder settings modal:
- Time picker (15 min before, 1 hr before, 1 day before, custom)
- Email / SMS toggle
- Preview of email content
- Save button that returns { time, method, preview_ok }

Use shadcn/ui Dialog + shadcn form components.
Export as single .tsx file.
STEP 3.2 — Claude Code: Integrate + Test
> Take components/ReminderModal.tsx (from Bolt) and:
>   1. Wire it to the POST /tasks/:id/reminders endpoint
>   2. Add to TaskCard component
>   3. Write Playwright E2E test for the full flow
>   4. Run vitest and playwright, fix any failures
STEP 3.3 — Claude Code: Review + PR
> /review-pr
> /commit
> Generate a PR description including:
>   - Summary of changes (per module)
>   - Test plan (what to verify)
>   - Rollout plan (feature flag)
>   - Observability (new metrics)

QUIZ

FINAL
CHECK

Kamu ship PR untuk feature baru, dan CI failing di integration test. Urutan tool yang paling efektif untuk fix?

A. Bolt — regenerate entire PR from scratch
B. Claude Code untuk investigate (CI log + code), Cursor untuk fix targeted files
C. Copilot autocomplete — cepat di IDE
D. v0.dev — prototype a fix

Jawaban B — Claude Code paling kuat untuk "investigate dengan konteks" (baca CI log, trace ke code, formulate hypothesis), lalu Cursor untuk apply fix yang targeted. Bolt/v0 (A, D) tidak tepat untuk bug fixing; Copilot (C) bagus untuk line-by-line tapi kurang powerful untuk investigate root cause.
